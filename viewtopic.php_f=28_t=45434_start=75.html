Mace_166

Tue May 08, 2007 12:05 pm by Mace_166

I have mentioned this recently and thought that i should make it clear to help rhose who don't fancy trolling through the pages of suggestions and bug reports.
The main cause of crashes in the current release is due to some dodgy encounter files. They are called, "tradep_trade_transport.ini" and "tradep_miner_transport.ini". The first one is by far the most common as i haven't added the miner one to any new systems. It is for trade patrols with transports.
I'm not sure when this started not working, but there is a very simple solution. As all of the ships that have transports also have armoured transports, we are going to make them spawn instead.. To do so:
01. Copy the contents of tradep_trade_armored.ini
[EncounterFormation<--close bracket
ship_by_class = 1, 1, sc_armoreds
pilot_job = armored_job
make_class = class_patroller
ship_by_class = 0, 1, sc_fighters
pilot_job = escort_leader_job
ship_by_class = 1, 2, sc_fighters, -1
pilot_job = escort_job
formation_by_class = gunboats
behavior = patrol_path
arrival = cruise
allow_simultaneous_creation = yes
zone_creation_distance = 0
times_to_create = infinite

[Creation<--close bracket
permutation = 0, 5

.. into the problem files and save.
This will mean that you will see armoured transports instead of transports and will not crash. The area transports around bases is ok, so leave that one the way it is.
What this changes
This will work fine in both single player and multiplayer. For multiplayer, the encounters are all run server side, so they will be used and any differences will be ignored.
If a server runs these changes, all clients will gain the advantages even if their files remain the same as the current release.

Feel free to post up any comments.
