Ogu

Thu Oct 26, 2006 4:02 pm by Ogu

Basically, your script was a little screwed up. You have </header> at the bottom of your script file - it should be after </options> nearer to the top. Also, scriptversion is usually 1.3, not 1.0, but I'm not sure if this makes a difference.

All of your <exe file="exe\newplayer.fl" method="sectionreplace" options="1:0"> should be <data file="exe\newplayer.fl" method="sectionreplace" options="1:0"> and the </exe> should be </data>, for all entries.

You're missing
<dest>
money = 500
</dest>
in your bottom data entry (below </section>.

All your [player entries should be [Player, with a capital P (everything is case sensitive).

One more thing...my newplayer.fl file is encoded, so I'm not sure if your file is different.

Alright. You see how you have 3 options (Normal, High Class, Real Rich Class)?

Each one is editing the same line, so when you try to activate the mod, the first option makes the "money = 500" line become "money = 0". Then the second option tries to find "money = 500", but can't, so you get an error.

You need a "default" option in each of those - meaning it just leaves the cash as "money = 500", For example, under <Options>, you would see:
<pre><font size=1 face=Courier>
;Option 1
<option name="Starting cash?" savesafe="true">
<item id="0" name="No Change/Default">
</item>
<item id="1" name="10,000">
</item>
<item name="100,000">
</item> </font></pre>
And so on for as many as you want.

Under the data part, you'd see this (this is for another mod, but you get the idea):
<pre><font size=1 face=Courier>
;Option 1
<data file="data\missions\m13\m13.ini" method="replace" options="1:1">
<dest>
Act_AdjAcct = -500
</dest>
<source>
Act_AdjAcct = 10000
</source>
</data>

<data file="data\missions\m13\m13.ini" method="replace" options="1:2">
<dest>
Act_AdjAcct = -500
</dest>
<source>
Act_AdjAcct = 100000
</source>
</data> </font></pre>
Notice how there's no "Option 0" (1:0) in the data part? That means that if you choose "No Change/Default" from the drop down menu, no changes will be done. You need to make an "option 0" for each of your 3 cash Classes - just don't put anything in the data section.

Then you just pick ONE of the values from ONE of your three main options to make it work right. If you still have problems, just make one big list with all the different cash amounts (from 0$ -> $999,999,999). Hope this helps a bit.
