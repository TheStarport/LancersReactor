Vast majority of reasons that prevent flserver (FL's server program) from contacting the global server and thus be listed is due to DirectPlay being messed up. DirectPlay is part of DirectX and you should have the latest version, v9.0b. When in doubt? Reinstall DirectX.

Other reasons involve gateway's or ISPs (especially on college campuses) blocking certain ports because they think you are P2Ping or running a server. For connecting to the global server I believe there is 4 or 5 outgoing ports flserver opens. If any of those are blocked you could have problems.

If you want more help we (the TLR community) needs a LOT more about your setup than just two sentences.

Start Menu -> Run -> enter Dxdiag -> click on the networking tab
Under this tab, there is a box at the bottom of the window, if it says no problems, click on 'Test Directplay' and see if it works, if it doesnt its probably a couple of things:
A.) ISP (or something of the like as said by earendel
B.) Firewall/router
C.) DirectX
Now, if on that bottom box it says it has detected problems, you need to install 9.0B, or reinstall it.
