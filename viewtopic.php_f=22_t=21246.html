''Flstat'' Queries...

Hey, I'm building the **UK Server** website and we've decided to try out the 'Flstat' program that displays players statistic on a particular server, which is available on the downloads section of this website.
I tested the program out on my computer with my own make-shift server with only one user. Worked in an instant and brought up a lovely table with the users stats.
So anyway, after I'd set it up how we wanted it, I sent it across to the guy who runs the server. He has alot more users, over 2000 now, and the Readme files says it does take a while and put a heavy load on the server, but we've tried leaving it for 2 hours and it still hasnt finished.
Should it take this long to process 2000 accounts? If not, then what are we doing wrong?

Cryogenius

Sat Jul 12, 2003 6:34 pm by Cryogenius

Cogen,

Firstly, make sure that you are using version v1.0a , as the previous version 0.9 had a bug that would cause it to get stuck on accounts that have no characters.

I would expect that a Pentium III would take around 2 minutes (not hours) to do this may files, longer on slower machine. If it is still taking longer that you expect, then perhaps you have a corrupt character file. The counter in the window should be increasing at a steady rate.

Hope this helps,

Cryogenius.
