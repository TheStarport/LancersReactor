genedefect

Tue Apr 29, 2003 11:43 pm by genedefect

On my 3.06 Ghz I use for my server, I am running about 50% CPU with 45 players.

Now at first glance it would appear that 30mHz per player is needed, but I have also found that just running the server takes up about 15% of the CPU, even before I get players logged in. Also, I am running Hyperthreaded which the FLServer does utilize.

My best estimate is a 450mhz to 600 mhz just to run the server with a small group of friends. After that you are looking at about 20mHz per additional player.

Now, don't try to run over 45 players on a box until the patch comes out. At 45 players my server is only using once again, 50% CPU, but it is also only using less than half of my upstream of 768k. If I up it to 50 players, the FLServer application itself starts to freek out and usually takes itself down shortly there after.

If you have anymore questions, let me know.
