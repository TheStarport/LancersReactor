Ok, here goes:

Blood Dragon:

Firstly, in Goods.ini, add these entries:

[Good
nickname = BD_hull
category = shiphull
ship = ku_dragon ;Blood Dragon Ship
price = 63200 ;Put your price
ids_name = 12006 ;Put your IDS Name
item_icon = Equipment\models\commodities\nn_icons\ku_elite.3db

[Good
nickname = BD_package
category = ship
hull = BD_hull ;Referance to the section above
addon = ge_ke_engine_01, internal, 1 ;These are default from the original Dragon, you can put in whatever equipment you'd like.
addon = ku_elite_power01, internal, 1
addon = ge_s_scanner_01, internal, 1
addon = ge_s_tractor_01, internal, 1
addon = shield01_mark04_hf, HpShield01, 1
addon = LargeWhiteSpecial, HpHeadlight, 1
addon = SlowSmallGreen, HpRunningLight01, 1
addon = SlowSmallGreen, HpRunningLight02, 1
addon = SlowSmallGreen, HpRunningLight03, 1
addon = SlowSmallGreen, HpRunningLight04, 1
addon = SlowSmallGreen, HpRunningLight05, 1
addon = SlowSmallGreen, HpRunningLight06, 1
addon = SlowSmallGreen, HpRunningLight07, 1
addon = SlowSmallGreen, HpRunningLight08, 1
addon = SlowSmallGreen, HpRunningLight09, 1
addon = SlowSmallGreen, HpRunningLight10, 1
addon = contrail01, HpContrail01, 1
addon = contrail01, HpContrail02, 1
addon = DockingLightRedSmall, HpDockLight01, 1
addon = DockingLightRedSmall, HpDockLight02, 1

Next in market_ships.ini, find the basename of where you want to put it, and add this line at the end:

marketgood = BD_package, 1, -1, 1, 1, 0, 1, 1


There, you can now buy your blood dragon wherever you put it, and it's fun to fly!

Next up: Bases that look like jumpholes.

First, open up your decrypted SolarArch.ini file, and add this block to the end.

[Solar
nickname = jumpholebase
ids_name = 60211
ids_info = 66146
type = STATION ;IMPORTANT TO HAVE THIS LIKE THIS, This tells the server to send the player to a base, not to another system.
DA_archetype = solar\dockable\jump_hole.3db ;Make it look like a jumphole
material_library = solar\Solar_mat_misc02.mat
mass = 10000.000000
loadout = jumphole ;Gives it the SFX, for the jumphole, the essance of what a jumphole really is.
solar_radius = 600
shape_name = NAV_jumphole
docking_sphere = berth, HpDockMountA, 150 ;Make sure that you have a berth, this allows small ships to dock, if you leave this as JUMP, then you'll most likly CTD. You can also make it "moor_large" "moor_medium", or "moor_small" if you want mooring capacitys
hit_pts = 999999961690316250000000000000000000.000000
phantom_physics = true ;Allows you to fly through it.


Now in any system, simply create a base, and change the archtype to jumpholebase. For example, if you replaced the old section (Li01_08) in new york...

[Object
nickname = Li01_08
ids_name = 196773
pos = -35763, 0, -25450
rotate = 0, 40, 0
Archetype = largestation1 ;NEWARK STATION
...(more in-file, not needed here)

and changed it to

[Object
nickname = Li01_08
ids_name = 196773
pos = -35763, 0, -25450
rotate = 0, 40, 0
Archetype = jumpholebase ;Note the only change is here
...

Then Newark would now look like a jumphole, but still work.

And finally:

Dock at A, Launch at B

Simple. If you look at Newark's info in Li01.ini, you'll find:

[Object
nickname = Li01_08
ids_name = 196773
pos = -35763, 0, -25450
rotate = 0, 40, 0
Archetype = largestation1
ids_info = 65761
base = Li01_08_base
dock_with = Li01_08_base ;THIS IS OUR LINE!
voice = atc_leg_f01
space_costume = br_karina_head, br_karina_body
reputation = co_ic_grp
behavior = NOTHING
difficulty_level = 3
loadout = space_station_co_01
pilot = pilot_solar_easy

dock_with is the line we want, if we were to change it to li01_01_base, then you'd dock at Newark, but come out at manhattan.

The only problem with this is, when you dock at newark, the manhattan base is loaded, making it impossible to view the true newark base.
