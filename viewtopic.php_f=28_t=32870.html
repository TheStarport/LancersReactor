Thu Aug 12, 2004 4:55 am by Black Eagle

Okay, I'm back from vacation!

Nice to see that so many people like my mod. Thanks to all of you for playing it and for reporting the bugs and your ideas:

King Lin:

You are right. You can't buy the ships, because of the other mods. If you used FLMM, you should have seen, that they conflict with my mod. They also overwrite the file, in which it is said, that for example Planet Primus sells the Horus.
So, please do not activate any mod taht conflicts with my mod in the FLMM, otherwise you surely will encounter such problems.

Many thanks for all these ideas. Some are really good and I maybe will include them in the next version. Thanks again and have fun, playing the mod.

Kwitzak: Sorry, but I do not have so many time to mod, and the time I get I spend to make the next version of the Monkeys. But you should be able to take the Monkey Freighter from my mod.
At first, if you want to release an own mod with this ship, you have to ask baene for permission, as he is the creator of the model. And please give him and also me credit if you release this mod.
Now, for getting the freighter out of my mod.
You need the following files:

shiparch.ini (in Data/Ships)
monkey_freighter.sur (in Data/Ships/Monkeys/Freighter)
monkey_freighter.cmp (in Data/Ships/Monkeys/Freighter)
monkey_freighter.tga (in Data/Ships/Monkeys/Freighter)
engine_equip.ini (in Data/Equipment)
misc_equip.ini (in Data/Equipment)

if you want the monkey freighter shield:

shield_equip.ini (in Data/Equipment)

You should know, that in these files are also my other new ships. You need only the parts with "monkey_freighter" at the top. For example in the shiparch.ini:

[Ship
nickname = monkey_fr (this stands for freighter)
archetype = xxx
...

Then you should be able to get the freighter out of my mod.

If you still get problems, please look here in the editing forums, how to include a new shipmodel in the game. You will find several guides that can help you getting the ship out of my mod. Hope this helps!

Gord and W4lt3r: This is not really a bug. Please make sure, that your whole Freelancer directory is NOT write protected. This bug occurs, if your exe folder in the main Freelancer folder is write protected. If you still get problems, please download and install the FL SDK 1.3 and the Unofficial FL Patch 1.4. This should really fix the bug. You can download these tools here on lancersreactor. Have fun!

Mace Nosam: I used lots of different tools and it would take much time to write them down here. But there is a thread here in the editing forums:

Go to the editing utilities forum and look in the thread "Utilities, Tools and other goodies". There you should find all the tools needed for editing with a short description.
Now some tools I needed very often:

FL SDK 1.3 (all ini files, successfully decoded and ready for editing)
FL Ed-ids (to make your own new names, descriptions and texts appear in the game. For example a ship description)
FL Explorer (I used it only for the orientation and position of the objects in the new systems, because it has some bugs)
FL Ini-analyzer (To find bugs)

But there are many others, as you can see in the thread i mentioned above.
I hope I could help you and would be glad to see a mod made by you. It's everytime good to see new modders. Good luck!
