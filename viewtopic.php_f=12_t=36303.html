Your firewall won't have an effect on ping times. It's more likely to be a hop between you and the server in question that is pining badly or worse giving packetloss.

So get an IP tracer and run a ping trace to the server while you do absolutely nothing online via your pc or anyone else on your lan for that matter. So stop all auto updating virus scanners/windows updates etc and then run something like http://winmtr.sourceforge.net/ but best to set it to scan every 5 seconds so you don't piss off the server admin and get your IP blocked.

Once you have run a trace for at least 10 minutes you will easily see if there is a routing problem, then can find out if its your ISP's routing or other. If it's your ISP then they can help, if it's not theres then you complain to who ever owns the router that is pining badly.
