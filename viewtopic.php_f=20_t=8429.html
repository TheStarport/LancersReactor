I ran into something "similar" to this and this was the way I fixed the problem.

Every time I installed a new version of the game, be it beta version(s), Gold, or retail on initial installation I got that popup window showing my account ID. I did a screenprint of each one. Thank god for following directions!

Now I used the switch feature of WinXP and when I installed the retail version of the game on one user, I got that screen with my user ID, did a screenprint and no problem, installed without a hitch. When I installed it I chose to make the game available to all users too.

When I went to the next user and tried to start the game I got that same account ID screen and thought that was weird (different ID) and made another screenprint and then hit "create." Well, it said wrong ID. So I quickly unistalled the game, and went back through the exact same steps. First installation WAS the same ID I saw before on the first user on the first install. When I got to the 2nd user it did have a different ID than the other user so I simply erased the ID, grabbed my screenprint from the installion on the first user and bingo, it initiated just fine.

Somewhere in this post is probably the answer to the problems people are experiencing.

In addition, this was right on all the screenprints:


There is a shortcut on Freelancer start menu that will display you ID for printing or copying


You know, START/ALL PROGRAMS/MICROSOFT GAMES/FREELANCER
