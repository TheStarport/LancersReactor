I have a DLink DI-704 and it is the best but, er, OK. Pertaining to the Linksys BEFSR41, did you try the default login? It is no user name, password admin. This is public domain info here, (flip the page to read it). This is why as Zaphod says, change it immediately!

You then should set your server a static LAN address. Why? So you can make permenant tunnels for DirectPlay which is part of DirectX which flserver uses. You need only one port, any port, not necessarily the range that FL suggests which is 2302-2304 but you'll probably have less conflicts if you use one of those ports. The second tunnel is for FLAdmin which is used to remotely control your server. That, too, can be any one port. Just remember it along with your IP address when you attempt to log on from afar. Unfortunently your Linksys doesn't allow you to set this up (DLink does) but you can still set a static IP LAN address in Windoze (see pg 75 here).

In the BEFSR41 you will have to redirect all incoming data on those ports to your static LAN address by creating two tunnels. In the Port Range Forwarding screen (page 65 of your manual here) is where you'd make those tunnels. I noticed it mentions you'd have to turn off DHCP to get port forwarding to work. I don't see why if you have your server set to a static LAN address but I guess that's Linksys for ya! (others, please feel free to correct this interpretation)

More good advice from Zaphod, use Lancerguard as you'll have lag problems from modders if you wish to play mod-free. Then all you need are players!
