actually the best, and only way IMO, to do this kind of massive edit is with the UTF>XML XML>UTF tool available on the EOA consortium's website - it extracts and tags all the data, which can then be easily previewed and modified.its important to get the right audio format - freelancer uses the official fraunhofer MP3 codec( but the files use the .wav extension*shrug* ) which is kinda annoying,since some codec packs contain the hacked/reverse engineered royalty-free codec which will break the game.... you can use a regular PCM wav afaik, but the file size is way too large

Cold_Void

Fri Mar 24, 2006 12:21 pm by Cold_Void

if you are replacing every message, use the XML project. if you are replacing a few, use UTF editor to do the replacing. CRCtool is needed if you want to match a specific msg in the INI to a UTF sound file, so you could replace it in utf edit or find its extracted&tagged file - using the XML method you don't need to look in the INI's at all if you want, just listen to the file,make a new one and save over it.

the message system is big, and if you don't know the difference between a UTF CRC id and a decimal CRC and hex CRC, you definitely want to use the XML method unless you're willing to learn...mostly on your own
