Well I'm not going to pretend I have all the answers but I'll give your problem a try.

I, too, run FLAdmin. Good program but I don't have a reason to use the remote control as my site hasn't become too busy. As for starting up multiple flserver programs, it would seem for some reason FLAdmin doesn't pick up the fact flserver is running. I have never used the server operator so I won't relate to that program but the fact that this issue happens with more than one program points squarely to an OS issue. I beleive FLAdmin looks for an OS provided list of running programs. The reasons for not picking up the running flserver could be many. One I am thinking about is security and permissions. Another is ownership. What OS are you running? I had to go to WinXPPro because Win2k didn't work with flserver by not connecting through the network (I had installed SP3 though, which people say is the problem). Are you starting FLAdmin as a service? Is everything running in an admin account? Just some thoughts.

FL account browser is written by akataka and can be found at www.flserver.com/downloads. That is if he has posted the latest version yet. This file calls home to decode the files but that's it. He has another tool called FL connection monitor. It uses the accounts.xml file generated by the account browser to identify people who are listed by IP (places a character name to the saved game file name). Akataka did beta testing for FL and continues to work with the Digital Anvil team I beleive.

I have heard of one other program that can observe the accounts in real time but I misremember the name. There is also FLStat which is a tool for extracting statistics of each character. Many are posting the generated lists to show highest kill totals, money totals, etc. Cryo the author is also working on linking different websites with these stats to get a super stat list. Myself? I would like more real time tools that can edit as well.

My server follows TLR rules (but it's not affiliated yet, do you see Stinger?). So I run LancerGuard which is installed on the server side, it modify's equipment files and prevents some pretty abused cheats (otherwise known as mods on a modded server). When it detects a cheat it boots the player like the flserver patch does. Unfortunently It doesn't block cloak and speed hacks. But every little bit helps for an unmodded server.
