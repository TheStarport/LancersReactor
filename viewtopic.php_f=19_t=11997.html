Two Freelancer Servers

Greetings,

This problem is similar to several already posted on this forum but this has a little twist in it. I will copy-paste what I have written in a newsgroup elsewhere.

"I have been running a server for a while and have built up quite a good
character for myself. My friend has also started up a server and I would
like to transfer my character to his server (and vice versa).

I tried to do this, but I noticed that the folder names are randomly created
(not by player ID). Also, if I transfer all the accounts over, only one of
the servers will appear online. How can I properly transfer the character files
and have both servers appear online?"

It seems that when two server have the same accounts on them only one (the one with the original accounts) will appear online. Does anybody know how to fix this yet still be able to transfer data between the two?

Thank you

If you copy all the files, you've essentially cloned the server. If both are running at the same time, players will effectively be connected to one or the other randomly.

I've moved individual characters this way:
1) Get a listing of the MP account directory on the new server.
2) Have the player whose character(s) should be moved connect to the new server and create new characters with the same names as their characters on the old server.
3) Find the newly created account directory by comparing a fresh listiing of the MP account directory against the old listing. Note the names of the files inside the new directory.
4) Find those same named files in the MP account directory on the old server. They will be in a different account directory, but the files will be named the same.
5) Overwrite the same named files on the new server with the same named files on the old server.
