Mace_166

Wed Aug 09, 2006 11:58 am by Mace_166

Right, got it. In order to play single player you have to install the multiplayer mod first and then activate the opensp. The opensp is just a script and a few very small changes that do away with the story.
So, to get playing, i'll do the whole order:
01. Install SDK v1.3
02. Install FLMM 1.3 (might work on higher versions, i haven't tested)
03. Install the Frontierspace 1.1 multiplayer.flmod file. If you double click on the .flmod extension it automaically loads it into the FLMM start up list.
04. Install the Opensp files for Frontierspace
05. Close FLMM
06. Re-open FLMM again (just to avoid problems with it not updating)
07. Activate Frontierspace multiplayer (it might take a while)
08. If you want opensp, activate the opensp too and choose the options. Don't choose the starflier, there is a buggy loadout for that one.
09. Delete or move any saved games, the autosave and restart.fl from the saved game directory (their location is selectable from the FLMM drop down menus)
10. Right now you can play.

You were activating only the Opensp with means that you were spawning with faction and base entries that are not compatable with original freelancer.
You might want to have a brief look through some of the bugs found on this thread. Basically, New Liberty (Boston/Tenessee/Chicago) are a bit buggy, as is Beliati, California and quite a few others.
The good news is that the next version is not too long away.
