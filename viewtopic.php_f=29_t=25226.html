Okay - a few extra things about this file.
The Mbases.ini file is very very pissy, in fact, if you are using one of liberties existing bases and the missions aren't working, then the chances are every base AFTER that base won't be working either.

Simple things like errors in the base number (ie putting Li01_08_base instead of the one you mean, or even leaving the old base number in it when no longer there) will cause missions and bar people NOT to appear. Other things that might cause it are, well from yours

offers_missions = 0 <---please tell me why that is there for your base faction.

Do you see NPC people inside the bar (apart from the barman), are they there? Do they offer things as well?

You might have got rid of a storyline character as well - which means it might get pissy with you too. Make sure all NPC names are correct and tie into the ones at the top (ie base faction part).

It doesn't matter WHAT the npcs are actually called there - as long as the ids name is correct, then you don't have to call them the faction in the nickname - that makes no difference.
so nickname = bob will work as long as
npc = bob for one part.

Now - when editing NY stuff you really have to be careful as alot of it is involved in the story etc.
However, try the mission = 0 part first. Also - its a great way of finding out which entry is b******** up by going to other bases and checking them. That way you find out at which point in the mbases.ini file the problem is. Check the 9th base in the system - or the junkers (13th) - does everything work there? Then if it doesn't you comment out the base for li01_08 and then check again - anyone there now? If so, you find what is causing the problem quicker. This helps if you do lots before checking.
