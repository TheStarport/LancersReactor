ship model "spoofing" other ship problem

I am hoping someone has run into this before and can help because it has me totally stumped.

Two new ships in the game. Both work perfect, look perfect, have accurate info cards. Put one in a system with lots of other ships and everyone can see each one by itself without issue.

Put them both in a system together, and that is when the "spoofing" begins. If ShipA enters the space first, when ShipB arrives it will have the appearance of ShipA while retaining it's true stats and info card. The reverse is also true - if ShipB flys in first, then when ShipA arrives it will look like ShipB. This condition is viewable by everyone in the system.

My first idea was perhaps shared .sur files, so I replaced ShipA with an original .sur from one of the standard ships, but it had no effect on the way the ship was drawn. (I'm pretty sure they are not possessed by Nomads, although that was my second thought)

If anyone has suggestions on what might be causing this, I would greatly appreciate hearing from you. Thanks.

when you exported them from milkshape you used the same names in regards to the hull group and/or also the textures. I've never seen this with the entire ship, but have in regards to textures. Though the same should be applied, the freelancer crc calculator generates names based on the groups, the top most mesh is used as the name for the entire ship inregards to what the crc id calls it as, thus when it calls forth for a ship which is of hyptothetically 15459874 and there is another with that same identification number it thinks it has already read it in, and thus does not re read in the new ships data. Again I have only seen this in the past in regards to having textures which have identical names between ships but the actual picture being different, but from what you described this sounds very similar to what happens with textures. So just rename the mesh groups of one of the ships and re export.
