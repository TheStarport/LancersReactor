Bejaymac

Thu Oct 19, 2006 11:45 am by Bejaymac

Just mailed a fix for you.

**EDIT**
This is for those that get white ships but don't know how to fix it.

Afaik there are two causes of this, the first is that the graphics drivers and Freelancer aren't 100% compatable. I got FL when it was first released and had no problems with it, 6 months later I found this site and started downloading some of the mods, this was when I found my first white ship and despite a trawl through the forums I couldn't find an answer(I was a major lurker back then).A week or two later Intel released a new set of drivers for my chip set and all of the white ships had color.

The second cause is using a file type that FL recognises but uses an internal format that the game doesn't support. FL needs them to be 24bit RGB .DDS or .TGA(indexed TGA works for some parts of FL), you can use 32bit RGB if your messing about with Alpha channels. TGA's need to be uncompressed and flipped vertically, DDS wants to be DXT1, DXT3 & DXT5, some of the other DDS formats are supported but those 3 are the safest for general use. Any format that milkshape allows (jpg, bmp, gif etc) will be exported into the MAT file with a tga tag by the exporter but they will still have their generic format with the end result being a white ship in FL. This cause will catch a lot of people out especially if they're using Hardcmp, unlike FL Hardcmp just isn't fussy about what it shows for a skin, if your drivers support a format Hardcmp will show it.
