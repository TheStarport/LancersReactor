Darkness2k

Fri Mar 14, 2003 6:39 pm by Darkness2k

I am talking about Multiplayer
IcE runs the server, therefore IcE has all the multiplayer accounts on his HD already. All he has to do is copy the .FL file from what ever he finds is the right account directory and put it in the new account folder that his friend has made.

I have already copied accounts from one server to another, I know it works... for me at least

Heres a basic example: -
"My Documents\My Games\Freelancer\Accts\MultiPlayer"
Here there will be a folder for every member thats ever logged into the server.
Lets say there are currently 2 folders.
36-6b23df12
63-21fd32b6

36-6b23df12 is the account that his friend originally had...
His friend makes a new account with his different Account ID
so a new folder is created, called (for example) 45-375ff34a
All IcE has to do, is (with his friend logged out and the server no longer running) copy the contents of folder 36-6b23df12 and paste them into 45-375ff34a.

When the server starts again, his friend should be able to log in with his old character with the new account.

Darkness2k

Well one way, is to find out the last time his friend logged on and find the file that matches that time...
another is trial and error..... depends how much time you have lol

The easiest way to check to see if you've moved the right file is by using the FLServer program

Run it, get it to start hosting (turn off net access for now so you dont get people logging on)
Go to the Edit menu, then Accounts
It lists the character name under than of the various ID's... you cant match the IDs there to the folder names unfortunately..

So try copying one account into another folder (don't overwrite anything) and see what gets moved.... do that till you get the right one and delete all the accounts that are not required...
He should then just end up with his character and no one elses!

I know, its not the most efficient thing to do.... but its better than losing your character altogether isnt it?
