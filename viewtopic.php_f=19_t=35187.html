Sun Oct 03, 2004 5:10 pm by Earendil

OK first, it has been said many times but this is how it is. Do not install the patch on a client only installation! Unless you run a server the patch may do some weird things to FL.

Next, you're playing FL which means you are a client. It means you have no need for INCOMING ports which is what all those firewalls block unless specifically told to open them. Playing FL as a client uses about 10 OUTGOING ports which your firewalls all let through (ZoneAlarm is the exception). In my experiance every firewall, both hardware and software, blocks INCOMING ports only except for ZA which blocks both INCOMING and OUTGOING ports.

Last, what is strange is what you see. You see a server list but no pings and can't log on? This infers you can reach the FL global list server but can't initiate pings from your own system? To see if that is true try using the command line program ping to ping some website with your Internet connection on. As for the log on issue, it may be a specific DirectX issue. Certainly all logon issues with the global list server and other FL servers have a common usage in DirectX.

So, I suggest you uninstall, reinstall FL. Do not install the patch. Install/upgrade to DirectX v9.0b (I have heard their may be issues with v9.0c and FL, certainly I had issues for a while when I got SP2 which included DirectX v9.0c [I think but it could have been an overclock problem as well! ). Let us know how it goes. Good luck.

WAN_SOLO

Wed Oct 06, 2004 12:46 pm by WAN_SOLO

If you are running any software firewall turn it off. Make sure you check the firewall that is built into XP. Service pack 2 seems to turn it on by default. Also if you have a router that you go through pull it out of the way and hook you comp directly to your modem. remember you will most likely have to powercycle the modem to get it to recognize the new connection. If you connect fine with out the router in the way then that is where you problem lies and you will have to sort that out. If while you are directly connected you still are unable to get the full global server list then you have a software issue and you need to look for anything that blocks directX or anything that acts as a firewall (such as XP firewall, black ice, zone alarm, etc....)
