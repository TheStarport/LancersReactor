Thanx NEO for hosting a RB server. I know the servers crashes quite often in MP unfortunately, but i'm working to fix the issues in EFA such as the commodity errors and too high ships density in planets areas. Hopefully this will stop the servers from crashing.
Superchan, the issue with the poison, toxins and other mass drivers are that they're coded not to do much shield damage. The default shield damage is indeed half of the hull damage so the ids's is NOT wrong. when you got 48/0 in weapons_equip.ini FL takes the default 0.5 shield damage and this is diplayed and used automatically in the game for that weapon. BUT, the mass drivers are a special weapons type that is coded in weaponsmoddb file to do almost null shield damage, here's the code for that:

[WeaponType
nickname = W_Driver
shield_mod = S_Graviton01, 0.075000
shield_mod = S_Graviton02, 0.075000
shield_mod = S_Graviton03, 0.075000
shield_mod = S_Molecular01, 0.200000
shield_mod = S_Molecular02, 0.200000
shield_mod = S_Molecular03, 0.20000
shield_mod = S_Positron01, 0.175000
shield_mod = S_Positron02, 0.175000
shield_mod = S_Positron03, 0.175000
shield_mod=S_CapSpec01, 0.1
shield_mod=S_CapSpec02, 0.05

so you see that's the cause why mass drivers do very little shield damage, not that 0 from weapons_equip.ini.
psychoscyte, the mod works in SP campaign with or without the patch, there's no reason to stop at level 4. I tested both versions and work perfectly in SP. Have you done a fresh install after starting the SP campaign? If you did re-install FL and the mod with FLMM 1.2 b10 everything should go smoothly. And pls do not use old versions savegames with RB 3.30, this will certainly cause weird errors.
