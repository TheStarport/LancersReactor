JoeBoomz

Thu Feb 05, 2004 9:16 pm by JoeBoomz

You can probably do it, but you may need more memory. A gig is enough for a single server, but I don't think it would be suffiecient to run two.

Here's how I would try setting it up. Haven't done it myself, so I can't say if it would work.

You'll need to run two separate user accounts in your Windows (to prevent playerfile overlaps) and user-switch to manage both servers. You'll also need two copies of Freelancer on your drive. Just install it once and make a copy of the folder, then you'll have your two copies of FLserver to use.

One of your servers you'll need to start with the commandline option /p2302 and the other will need to be a different port (like /p2303).

When players connect to your servers, they'll have to specify in their freelancer commandline option the port of the server they wish to connect to. Your servers may not appear on the global list.

I would recommend against running two servers. Keep it simple and just run one. Then you can appear in the global list, no one needs to worry about ports, and you won't have to limit your player max's on your servers. With that sort of horsepower you should be able to host a bunch of players without any lag (my guess is near 70).

Good luck!

-- JoeBoomz

Earendil

Tue Feb 10, 2004 1:14 pm by Earendil

I sincerely doubt you could run two flservers on one 'puter. While you might actually get the two to run as seperate programs (I have seen it happen by accident) they would have to use the same LAN connection and that is forbidden by the documentation (cannot run more than one server on a LAN). Of course the assumption here is you have a LAN between the server and the Internet. The only way I see this being done is if you could get two instances of EVERYTHING: flserver, TCP/IP driver, network cards/Internet connections, seperate LANs (if applicable). DirectX and through that DirectPlay can handle multiple instances games using DirectX on the same server but maybe not from more than one flserver executable.

In my opinion it may be more trouble than it's worth, IF it's doable. I suggest you get a honking, huge Internet pipe and run a 70 player server like Andromeda.

Well we did it, it was a pain in the patootie, but both servers are up and running.
The key to it was copying the FLserver install. I couldn't install two copies of the game so that was the only way. Thanks for the help on that Joe! The other thing that gave me a lot of problems was FLMM. Everytime I installed a mod on one of the servers, it would appear on the other. It took me way to long to realize that FLMM (as well as Flserver and IonCross) was being installed to the All Users Directory so everything I did was being duplicated to the other version of the game, event though they were on different user accounts.
They both appear in the Global list. one server takes the IP 69.56.135.35 and the other 69.56.135.34. They tend to bounce back and forth as to which server gets what IP. Troubling, but sometimes they get the same IP, but it doesn't seem to affect game play at all.
We didn't have to mess with the ports at all (thankfully). After we installed it and got IonCross and FLMM to talk to the right directory it worked no questions asked.

Thanks for the help and input fellas!
