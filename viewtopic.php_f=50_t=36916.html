Wed Dec 29, 2004 5:38 pm by forgottenlord

I read once that the max coords are -500000 to 500000. You'd have to do some manual editing to be able to design systems like that. In the data->universe->universe.ini file (save files as text), change a system's NavMapScale value. I believe (someone correct me if I'm wrong) that the max/min values are +/-(10000/NavMapScale) to remain on the HUD map. If you manually edit that early, Freelancer Explorer will use the new values and let you edit normally.

Sat Mar 05, 2005 8:17 am by Glock36

There is no minimum number of object that I know of but I am beginning to suspect that the max is 256. That is just my suspicion at the moment based on experience.

The navmapscale line in universe.ini below a system declaration is, in my own experience, not needed unless you want to make the system smaller.

the standard size of a system if there is no navmapscale line extant is 200000 x 200000 x 200000. if anyone cares to contradict, I am open to being wrong but this is my experience.

The bigger the number put in as the variable for navmapscale, the smaller the system.
I have no idea what the limit is but i'd tread lightly here and not make one smaller than the smallest standard shrink you can find in universe.ini
