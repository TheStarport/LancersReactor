Fri Aug 03, 2007 12:32 pm by bryanhbell

I too was able to resolve the problem of connecting to other Freelancer servers (both LAN and Internet) by uninstalling Norton AntiVirus 2007 from my Vista machines.

I also discovered that Vista's built-in IPv6 protocol presents a problem for Freelancer on local LANs. I was able to resolve this by turning off IPv6 in the Properties for my "Local Area Connection".

However, while turning off IPv6 resolves the problem for PC to PC connections on a local LAN, it doesn't resolve the problem when using the Freelancer server and client on the same machine, since Vista continues to use IPv6 for local-link traffic, even though you have disabled IPv6 for host-to-host traffic. To resolve that problem, I had to disable Vista's built-in isatap and Teredo network adapters in Device Manager (they appear if you choose to Show Hidden Devices).

A little Google research has revealed some information that to me indicates the reason anti-virus software and Freelancer might conflict on Vista is because of how anti-virus software interacts with Vista's built-in IPv6 services. For example, using the Norton Removal Tool in Vista will break the Teredo network adapter, which may indicate that Norton Anti-Virus is mucking about with IPv6 in a way that is causing problems for Freelancer.

http://www.computerhaven.info/forum/pri ... px?m=58023

Also, the following article seems to indicate that Norton Antivirus for Vista includes some functionality designed to prevent security problems with the Teredo network adapter.

http://m-net.net.nz/1535/security/news/ ... mantec.php

Further research may yeild a registry hack or some other tweak that will enable anti-virus programs to peacefully coexist with Freelancer.

First, thanks to everyone who have been posting to this thread. You really helped me clear up the issues I was having getting FL running under Vista. Good job to you all.

Next, there is a long term method for getting installers and programs to run without seeing the requestor asking you to allow the program to run.

You need to make sure the login you are using is enabled as an Administrator. Then you need to go into -->Control Panel-->User Accounts

When you see the window that shows your account and different things you can do, under the heading that says 'Make Changes to Your User Account' choose the very bottom option. It should say 'Turn User Account Control on or off'.

Set it to Off. It may ask you to restart, but thats unlikely. Once you have done that, you will no longer be bothered by requestors every time you run an installer or program that has permissions that Vista doesn't like.

That setting will ONLY affect the chosen account. All other accounts will continue to exhibit the annoying requestors until you do the same to them.

Hope that helps everyone!
