bakedpotato

Mon Sep 19, 2005 11:53 am by bakedpotato

The crashes may not be caused by the amount of players, it may be caused by characters that are modifying things and since the server files are not in sync with the player files this causes a crash.
If this is not the case you can of coarse limit the number of players when you start up flserver.exe you'll see a box with the number of players allowed.
By default it is set at 16.
Adjust accordingly until you find an the number that don't cause the server to crash.
You should be getting a yellow and or red icon on screen that is flashing giving you a warning.

The only suggestion I can give you is to watch when this happens and the player that just logged on is the culprit. Although the crash may not happen immediately, it happens when they use that modified piece of equipment.

Also if you're not already using the IONCROSS Server Operator program by Joe Boomz you may want to install and use it.

bakedpotato

Mon Sep 26, 2005 1:19 pm by bakedpotato

You can play Open Singleplayer games by installing the Freelancer Mod Manager to activate and deactivate MOD's.

Then you need one of the Open Singleplayer MOD's
there are 4 that I know of that just enable singleplayer mode.

Open Singleplayer Lite by Chips
OpenSP MOD v1.1 by Xerx
Open Singleplayer FuBi Edition by FuBi
Open Singleplayer MOD by Chips

You'll have to pick which of these Open Singleplayer MOD's that you want to use but they all work and all do different things. Read the MOD's description when you go to that MOD's download page.
If you don't know about the Freelancer SDK v1.3 files you may want to install the SDK files before you activate any MOD's. Some MOD's need the SDK files or they wont work.
Besides it cant hurt.
