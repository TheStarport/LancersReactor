Cold_Void

Thu May 18, 2006 7:20 pm by Cold_Void

sounds desperate; maybe i can help a little with some random thoughts - although i would be the last person to call myself a competent network expert.
so, prefaced with As Far As I Know:

freelancer needs a UDP outbound port range open as well as TCP inbound, 2300-2304 namely

Network Address Translation may be the problem - that happens when the forwarding,virtual servers,etc are not set just so

the gateway IP address listed in your LAN's TCP properties should be the same as the router's gateway, at least thats how mine works and how i fixed a similar listing-problem

windows also has TCP/IP filters in the LAN cards properties, and inbound/outbound should be enabled

DMZ has never fixed a connection problem for me, and we've all heard how unsecure it is

if you're using a DHCP setup try setting up your connection as static
re static setup:
subnet masks actually vary depending on the local network IP, so having them all set to use the same 255.255.255.0 is wrong and they should be calculated instead(google subnet mask calculator)

again, it might not be accurate info but maybe it will inspire some different approaches to your problem
