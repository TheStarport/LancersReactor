bRydeR

Mon Jun 23, 2003 2:36 pm by bRydeR

Yeah, I had a similar throught myself.

The server software would need to be re-written from the ground-up, but I don't see any reason why the client wouldn't support it. Guess it could get laggy with lots of players in the same area as each machine would need to keep track of the action, but a clustered or distributed server model could still potentially support a lot more players than the currently crappy server provided by DA/MS.

However, I foresee a number of challenges:
1. Legal - the existing branding and code would be Microsoft copyright, so you'd have to develop it without using or reverse engineering any of the existing coding and couldn't use the branding or possibly even the name "Freelancer" without Microsoft's blessing.
2. You'd have to work out what the existing server does without "de-compling" it - not a minor task!
3. The obvious issue of getting people with the right skills, calibre and motivation to dedicate time and effort to writing it.
4. The exisiting server uses the global server to announce it's availability; Microsoft own this server and would almost certainly prohibit a third-party from doing the same so it's be impossible to integrate with the existing architecture.
5. Uptake - you'd have to persuade people to commit resources to running the servers and players to connect to them which, without global server listings would be difficult to "market".

I think it's a great idea but I don't think the game has sufficient longetivity or support to warrant people dedicating the effort required to overcome the issues.
