FLConnection Monitor (FLCM) needs the file accounts.xml to link a FL MP saved game file to the character's name. This accounts.xml file is generated by AkaTaka's other program FL Account Browser (FLAB ) which is in beta and only a few sysadmins have (like me!). Unfortunently it does no good to give it out. I have typed this all before but I will do it again briefly. 1)AkaTaka hasn't said to give to anyone else and he seems to be afk 2)you need a unique user name and password for it to work and 3)the server FLAB uses to decode/encode the game files seems to be down.

What this means is FLCM will let you know which FL saved game files were accessed by players connecting to your server but you have to use something else (like JoeBoomz FL char ed) to determine their character name. Although all those programs are on my server and are up and running, accounts.xml hasn't been updated in weeks so many of the file names have no characters linked to them.

If you don't even see a file name then the IP address is wrong (if you have a LAN use the LAN address, not the internet address) or the port is wrong. The port is supposedly a spread from 2302 to 2304 but most people assume 2303. I set my server to 2401 so there is no mistaking what the port is.
