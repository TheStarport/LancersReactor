Rankor the THN's are just script files, you tell them to use items as they don't store the required info. The body animations are in the ANM files in the DATA\CHARACTERS\ANIMATIONS folder, I used these to make a THN for a female bartender about 18 months ago. The other animations are stored in the models CMP, open one with the UTF Editer and expand all of the sub-nodes under the animation node. The header & frames nodes under each joint map node is the actual animation, but even knowing this I still can't figure out what format it's in.

i found a method to recover the individual meshes - if you're willing to piece the ship back together like a puzzle, it could be what you're looking for.

open the cmp with UTF editor
delete all the 3db's but one- open the vmeshpart/vmeshref node and export the file to wherever, then delete the .3db tree and add a new node in its place, with a sub node- name them vmeshpart and vmeshref respectively and import the ref file back in.save the file and you (or me at least) can import the file with no explosion problems.

why it works? i don't know- its probably that the importer can't gather enough information for models with a LOT of meshes (sleeper ship, cityscape models etc)
what this does? just fools the importer into thinking the file is a .3db
