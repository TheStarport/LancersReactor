There is a hard way and an easy way. The easy way is to make the second server the same as the first and the second server goes away. By copying the Mulitplayer directory including the accounts.cfg file, this defines the MP server you are on. This accounts.cfg file seems to have the special MP FL server ID needed to decrypt file names, dir names, etc and is the server identifier. Yet two servers on the Internet with the same MP FL server ID depends on the refresh rate and time of refresh as to which server you will see on the FL global server list. Not pretty. (yes, been there done that)

The hard way is to take the file from the first server. Create a new character on the second server with the exact same name. Find the file name of the new character file. Rename and replace the new character file with the file from the first server. Now you are back in business because you didn't transfer the account, you transfered the character(s) only. Works between SP and MP.
