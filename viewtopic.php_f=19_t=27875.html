Lets Make Capture the Flag

Hello,

I know many people are dying for a version of this game for freelancer. Multiplayer gets old unless there is clan competition. We have deathmatch, but we need another type of gameplay. We need a mod that will allow "Capture The Flag" type of game play. I understand that we're limited to what we can do because of Microsoft's non-release of sourcecode, but here's what we need.

**Game Play**
-A system that has two planets. The flag is located near their home planet.
-Players must grab opponents flag and return it to their home planet by docking. This gives them a point.
-When someone dies, they lose the flag and it goes back to where you can retrieve it. In multiplayer, it isn't possible to drop items on death yet.
-System is clear of non-human players.

**Task List**
-creation of the system is easy. Having two planets and a sun is very easy to make.
-creating a "new" item, the flag, can be easily, but to have it respawn when lost? can we do this?
-if someone takes the "flag" to their homebase, can we "score" a point ?

I'm sure we can assign tsr programs or create a totally new variable set, but I hope someone is able to see how this can be done and make it. Please let me know.

Otto,

From what we have available to us presently, it's not possible, sorry.

The means I can use (scanning playerfiles) is limited in that the server only updates playerfiles when they log off, change systems, or dock/undock. With this in mind, it's possible to set up a loot container in space for the teams to find that contains the flag, but before an external program knows about the flag, the player carrying it would have to dock and wait a second for their file to write. Of course, in the interim, if the player died (or dropped the flag and no one was around it so it disappears), the flag would be lost. There is currently NO way of knowing the location of objects in FL space.

Sorry bud, but we're too limited by FLServer and we can't set up elaborate stuff without figuring out where information is stored in memory and trying to modify the system memory on the server itself.

-- JoeBoomz
