For Things to show on map

Asked a question about why a completely new object was not showing on map as a side issue to another problem on 3db file types some time back and didn't get a working answer. Finally sorted it after a little searching.

For something to show on map any object needs what is shown on map to be defined. This is in the solararch.ini definition of all objects which show up and is the line starting ----> shape_name =
For example ----> shape_name = NAV_surpriseX is the red cross usually used for wreck location.
If you look through solararch.ini you'll find the other types available. Don't know if u can make your own map "icons" yet , I'll have a look. Maybe someone else can explain.

Update - Might be talking to myself but someone else might want this info one day.

The files for the nav map icons are located in folder Data\Interface\Neuronet\Navmap\NewNavMap\Spaceobjects and are in 3db
format. You can make your own by using UTF edit program to edit a copy of an existing file. Import a new pic in tga format and change the texture name under Texture libruary to match your new pic. Rename your file to a suitable name and update your reference in solararch.ini for your object to match that of the texture name. As far as I know this is all you need to do it appears to work fine.
For More info on icons see the brilliant tutorial - Making Custom Ship Icons by Osprey. Ship icons are more complicated to sort out properely but the information is still relevant here.
