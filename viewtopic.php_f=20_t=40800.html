... hmmm ...
Your problem is afaik not a bug of the game (I think there is only one jumphole somewhere with htlm-leftovers). I have seen such mistakes in early mods - that was before the modders got some handy editing-tools.
Are you sure you did a reinstall exactly as described many times here on TLR ?
Example from Crossfire MOd:

Some MOD's need a clean install of Freelancer...
Especially if you have used any MOD's before you need to do a clean install of Freelancer.
By clean I mean make sure you get a nice shinny new set of files.

1st... Delete your Freelancer folder.
C:\Program Files\Microsoft Games\Freelancer
Delete your Freelancer folder instead of uninstalling Freelancer to make sure there are no files that sometimes can get left behind after an uninstall, unless you check to make sure there are no files left in the Freelancer folder after you do a uninstall...

2nd... Delete or move any previous saved games.
C:\Documents and Settings\your name\My Documents\My Games\Freelancer\Accts
Singleplayer and Multiplayer folders.
If you don't move or delete your saved games they can cause a conflict and will make Freelancer crash, guaranteed!

3rd... Install Freelancer from the CD.
The Crossfire MOD has the Unofficial patch by Buck Danny built into the MOD so you don't have to install the Freelancer SDK v1.3

If you don't all the steps above you will probably experience a crash or other problems.

4th... Then activate the MOD in the Mod Manager.

5th... Have a blast.


EDIT: Install FL SDK 1.5 as well

Argh

Thu Sep 15, 2005 4:57 am by Argh

1. Reinstall the game. Make sure to delete all savegames (My Documents/My Games/Freelancer/after that it's obvious- delete everything in the savegames folder).

The XML files are run through freelancer.exe, but one or more DLLs in the game controls how they're translated. I believe Engbase.dll is the key for English installs of the game. If this is corrupt, then it won't work right. We're talking about a parsing error, which may be either the main EXE or in the DLL that controls this behavior.

2. Check your machine for viruses, spy-ware and ad-ware lately?

3. Updated Windows lately?

Other than that... dunno, honestly. I suppose some other, legitimate package might interfere with XML translation, but I can't think what. I have lots of web browsers and other things on this box, but none of them have even slightly bothered FL. 'Course, I go out've my way to keep this box clean and fit, too.
