Never noticed this before when testing IFSO, but when using the option in IFSO to Reveal the map to Player for a character it seems to cause extra FLserver load problems.

Fixing it isn't easy, as you need to manually edit the FL character and remove all visit options from it. Or just get the player to make a new character and transfer all their equipment over.

So take note of this new minor annoyance with FLSO if and when you build new Admin characters.

The New Map info file for the IONCROSS Server Operator by Pinger is now in the download section...#1360...that's allot of files

This is a fixed version of the Map info file for the IONCROSS Freelancer Server Operator. If using the MapInfo file to give someone a full map that comes with IFSO it can cause extra server load, down to it adding single player Jumpholes, which have an adverse effect on the FLserver Load.
So this one just adds all the Online holes, note there are still some locked holes that can't be used, but no longer any of the single player holes that shouldn't show online.
The file is a self extracting exe that extracts to the current location, so put it in the IFSO folder and extract it there.
The default location for the Ioncross Server Operator is at:
C:\Program Files\IONCROSS Freelancer Server Operator mk.V
Enjoy! from Pinger
