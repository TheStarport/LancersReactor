Yes, need the answers to the good questions from Pinger and uk server. Although when FLSO is running and churning on my server through character files and lists, it takes up to 30% usage in small time amounts regardless of the amount of players. My server stats, an Athlon XP 3200 (2.2 GHz), 400 MHz FSB, 1 GB memory.

I have seen other reasons for this "mad CPU" usage. Occassionally I get flserver locked up with 99% CPU usage. FLSO thinks it's running so it's not restarting flserver. The odd thing is in the event viewer it mentions an event ID that can't be interpreted by the server. This implies something from another players 'puter was sent to flserver and it decided to churn on it full blast. It lists a DLL or other files of flserver as having the error which changes and the files listed are not consistant.

Other than that I have seen "mad CPU" usage with the pre-patch flserver but I think that may have been the symptom of the big memory leak. With the patch flserver's memory leak is much smaller.
