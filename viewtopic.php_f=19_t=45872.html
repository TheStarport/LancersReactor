Everyone that plays Multiplayer on their computer, whether playing by themselves or running a LAN game has to start flserver.exe
Meaning you can play Multiplayer games by yourself without having to log onto someone's server.
I have tested hundreds of Mod's in Multiplayer mode using one computer by first starting flserver.exe and then starting Freelancer and then playing without any trouble although I usually deleted the Freelancer folder and then re-installed Freelancer again to make sure there weren't any files left in the Freelancer folder.
And after re-installing Freelancer, installing the Quickfix files.

So you most probably have either some files that are causing the problem or possibly your firewall. If the problem is with the firewall sometimes just restarting flserver.exe and then Freelancer will work because now your firewall should be setup to let you access flserver.exe

Maybe if you gave more information, such as are you using a Mod, have you used and Mod's before and if you have, have you done a clean install of Freelancer - making sure that there are no files left behind in the Freelancer folder before doing a re-install.
Did you install the Freelancer Quickfix v1.0c after doing a clean install of Freelancer?

Rankor

Tue Jul 17, 2007 7:10 am by Rankor

I am still haunted by this particular problem at times when I am testing my works. The biggest reason is that the game is searching for a "Global List Server" and tends to get somewhat frustrated (poor choice of words) when it cannot find it. So it tends to crash at times. The only way I've mangage to get around this malady is to allow it to connect to the internet despite my not wanting it to do so. So, to keep unmodded players from logging on, I have to set up a password and limit the number of players to 1 (myself) . That seems to correct the problem. Until I can find a way to prevent the exe from trying to locate a Global List Server, I'll keep getting that error until I allow it to connect.

Other minor errors which the Server pulls up happen to be some of the infamous "Fate_Dissapear" , "Not_Equippable" , "Not able to insert base due to ID collision" , and "Failed to find" (even when the system does not exist anymore)

But in all, I think the biggest problem causing the crash is simply that the Server is not being able to connect to the Global List Server. I see this in my error log all the time unless I permit a connection with the internet.
