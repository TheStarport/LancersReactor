Tue Apr 27, 2004 8:03 am by warzog

As I've been unable to duplicate the crashes that people are encountering, I need some help to isolate the problem(s):

Several little ideas:
1) Try deleting the DLL's in the EXE folder-see if that helps.
2) Try deleting the Constants.ini & Initialworld.ini in the DATA folder.
3) If those don't help, try deleting the Scipt.xml file in the main folder.
4) And as a last ditch effort-delete the mission's folder-doing so will lock you into the normal SP missions.

Queries:
1) I've been running FL in straight XP mode-is anyone running it in an emulation mode?
2) I've been running FL without an update, or sdk v1.3 installed, is anyone?
3) What system(s) have you been to before the crash?
4) What base(s) have you been to before the crash?
5) What ship(s) have you owned prior to the crash?
6) What were you doing before the crash?
IE: Trading, Fighting, Salvage, Random Missions, Exploring, etcetera?
7) Commodities-getting reports of wierd pricing. I need to know what commodity, where you bought it, what systems you went through, and where the odd price was.
How long were you playing (non-stop) prior to the crash?

Also, if at all possible, can you email a copy of the FLspew.txt file from the game that crashed?
(FL creates a new FLspew.txt file each time you run FL. Even if you don't start a new game, or load a saved game.)
FLspew.txt is the file that gets emailed to MS if FL crashes, and it holds important information that can help to determine why this mod crashed on you.

In Windows 98, it'll be in the:
c:\Windows\Local Settings\Application\Microsoft\Freelancer folder.
In WindowsXP, it'll be in the:
c:\Documents and Settings\User\Local Settings\Application\Freelancer folder.

I'm not sure where it'll be in WindowsME, or Windows 2000.
Thanx!
