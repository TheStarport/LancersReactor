Tue Nov 11, 2003 6:51 am by Earendil

In my messing with saved game files I have found that the file name itself isn't important. It is as it was stated by Niwo, the directory names are unique and tied to the server key. The only way currently to transfer players in this case would be to match up each unique player's directories from each server and copy the contents from one to another. But the key is matching the directories from one server to a different directory name on the other. While any code could be written to copy the files, knowing (requires a lot of typing and would have to have new players constantly added manually) or decoding (specialized code for decoding directory names, never heard of any) the directory names is the hard part. I know of no software that does this automatically. I can do this with FLCharEd from JoeBoomz on a tiny scale, manually but that isn't feasible for so many characters.

Of course you can't run the same account IDs on two different servers. The global server gets confused. Similar to issue with more than one server on a LAN. FL is looking for only one flserver on a LAN (not talking internet), two confuses FL.

Would be an excellent idea though to extend an FL community through multiple servers! I wonder if JoeBoomz or AkaTaka can say more on this?

JoeBoomz

Wed Nov 12, 2003 6:39 pm by JoeBoomz

Hmm.. I haven't tried something like this myself. I think it's fairly safe for me to say you cannot copy your entire MultiPlayer folder to another server and expect it to work properly.

What I expect you CAN get away with is just copying all of your player folders to the new server machine. I do this on a regular basis with the IONCROSS server while testing stuff at home. I'll copy the entire multiplayer folder to my home pc, strip out the flserver.cfg and other files, and just have the actual player folders left over. My test server at home picks up the players without problems and I can log into my account on the IONCROSS server, then disconnect and log into the same account again on my test server. As long as you don't copy the FLServer.cfg file (which I'm sure contains your server name, news, and other configuration settings) and use a different .cfg file for your second server, I don't see why it wouldn't work.

Earendil, I'll look into re-enabling player re-naming in the next release of IFSO. I've just had too many requests for it, even though it can cause problems on occasion. I just have to double-check to make sure changing character names doesn't screw up IFSO either

One OTHER thing to try if it still doesn't work would be to copy EVERYTHING, then start up the server and just change your server name and news and that information, then restart FLServer so changes take effect properly. At this point, would it still mess up the global server list?
