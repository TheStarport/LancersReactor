Mystical GrAm

Tue Jun 22, 2004 7:05 am by Mystical GrAm

There is nothing wrong with that part Katastrophe. Just make sure you have added the base (as ew06_03_base) in the universe.ini and in the bases directory of your ew06 directory.

The ini in the bases directory should have the same name (ew06_03_base) and contain a text like:

[BaseInfo
nickname = Ew06_03_Base
start_room = Deck

[Room
nickname = Bar
file = Universe\Systems\Ew06\Bases\Rooms\Ew06_03_Bar.ini

[Room
nickname = Deck
file = Universe\Systems\Ew06\Bases\Rooms\Ew06_03_Deck.ini

[Room
nickname = Deck2
file = Universe\Systems\Ew06\Bases\Rooms\Ew06_03_Deck2.ini

It doesn't have the be exactly the same as here. Just that you have a bar and a deck will do.

Also make sure you have created the rooms. So go to the ew06_03 directory again, go to bases, then to rooms and add (if you use my example)

Ew06_03_Bar.ini
Ew06_03_Deck.ini
Ew06_03_Deck2.ini

If you haven't done that, then this is probably causing your problem.
To create them you could just copy them from Battleship Missouri in New York
(Li01_03_Base) and rename everything.

The part that in your original ini that is a bit weird to me is "space_costume = orillion, juni"

A normal entry would be something like "space_costume = pl_female5_head, li_female_elite_body" to use on bases occupied by the Liberty Navy. But you want the head of Orillion on the body of Juni? In that case it has to be "space_costume = pl_male4_head, li_hatcher_body"
But you will have a male head on a female body.

If you need more help, feel free to ask.

Check shiparch.ini file - you probs have this showing:


[Ship
ids_name = 237032
ids_info = 66566
nickname = li_dreadnought
LODranges = 0, 600, 800, 1000, 1500, 2000, 3000, 20000
msg_id_prefix = gcs_refer_shiparch_Libb
mission_property = can_use_large_moors

The moors don't work properly in FL, it never completes the dock sequence properly, as players were never intended to use them!

However, check for the "docking fix" - there are several versions, by far the most popular being the one Victor stated in using:
can_use_jumps

but this requires some editing of the solararch.ini file as well - so that stations have "can_use_jumps" in their docking port method part.

The other method by duriel is the older "can_use_berths" - which is what fighters use, but cocks up everything else and requires ALOT of editing in the solararch.ini file too...

However, you can also use the ghost docking ring and the battleship mod that comes with FLMM as well.
