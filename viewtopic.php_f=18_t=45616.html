Suggestion for Standardization

One of the biggest issues between mods is the amount of different information between them. We've got mods with new ships, new systems, new stations, new planets, new factions...etc, etc, etc.
This is a long shot suggestion since many people have their own ideas about things.
To make things more compatable in the future..
I would suggest an open library here that lists all important information.
This would be a list of all the items that sport an ids number. Since each number referrences a specific item or text..a "universal" "library" would be benificial that everyone could add to and also refer to when creating something new.
ids_names
ids_info
ids_info1
ids_info2
ids_info3

This idea is so if somebody wants to add a new ship (or whatever) to a mod, they can check this library for items that are already in use. This would prevent duplicate or conflicting entries.
Might I also suggest a library that includes all dll files for reference OR perhaps a set of UNIVERSAL .dll files that everyone can use in conjunction with the library of ids information?
