Hmmm...if one end is a VPN dial in (as opposed to LAN-to-LAN with different subnets) to the other LAN via PPTP and it's not working then I'd imagine Freelancer is looking for some broadcast packets which aren't being routed across the VPN. :/

No matter though: If you are both on ADSL anyway, why not start an Internet server and password protect it if you don't want others joining?

Alternatively, you could try modifying your Freelancer icon and add
-s<LAN-IP-ADDRESS>
on to the end of the executable, e.g.
d:\games\freelancer\exe\freelancer.exe -s192.168.0.10

When the client end looks for internet games it should show that server first?
(Note: I've not tried the above trick myself, picked it up from somewhere else)

HTH
