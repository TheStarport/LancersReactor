I think the player just has a custom faction listing, not listed under a title. I was thinking this because the freelancers rep is shown as individual faction listings in mpnewcharacter:

[Player
name = %%NAME%%
;initial_rep = %%FACTION%% <------- This is commented out in unmodded FL, because the rep is defined below as individual values like:

house = 0.91, li_n_grp
house = 0.91, li_lsf_grp
house = 0.65, li_p_grp
house = 0, br_n_grp

Under the initial rep, you can remove the semi-colon (; ) and place a rep there, like li_n_grp. The starting player will then start with the rep of that faction (be sure to remove the individual values first though, or the game will crash, having two rep definitions)

etc.

Yeh, and if you leave the %%FACTION%% but remove the semicolon, it will take the faction defined in newcharacter.ini. All those macros refer to that file, you can either directly set something up in the mpnewcharacter.fl file or use those macros and refer to the newcharcter.ini.
In other words, the player is not a faction "%%FACTION%%", that's just a macro to refer to a different place.

Okay....I got it....sorta....

This is what I can do. First off, I can't get the save game editor to convert files back for me, so I stopped trying to alter the newcharacter fl file. I can't seem to get the above idea to work, so maybe someone else can do better than I.

However, I have figured out, at least, how to be able to change Freelancer A1-1, to Navy A1-1.

I have OpenSp mod by Xerex running. Then open the initalworld ini. Using BinuQ, look at the first listing of factions. That should be Navy.

Move the [Group bracketed tag up into the cluster with the locked gate listings. Then move the nickname up as well, making it flush with the Group tag.

It will look something like this...

blah, blah, blah....Li01_to_Ew01_hole [Groupnickname = li_n_grp ids_name = 196846
ids_info = 66200

Everything needs to be flush. Save, and close, Reopen. You should see [Groupnickname=li_n_grp colored BiniQ-Green. If it's blue, it won't work.

Now, I've tried other factions, it crashed the game. So, from what I see, this is the only one that will work....but I'm sure you code wizards can find a better way, or avenue to get others to work.

As an insight, I recreated my accidental discovery of this while I was using the Faction creator tool. It was this tool that created this the first time as well. All I did with the tool was create the faction for the initialworld ini, and cut and pasted from the tool to the ini file.

Let me know if anyone else can reproduce this....I've been able to go back and forth putting it in to work, and taking it out with no effect to my saved games....
