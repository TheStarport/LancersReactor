Crash-Drive

Tue Apr 01, 2003 7:26 pm by Crash-Drive

This is something I have been able to deduce using the game data and some testing. There is a setting on wrecks in the .ini files for the various systems called "visit". This value is usually 16, but all of the wrecks in Omicron Beta and Omega-7 have a value of 0, and these wrecks can be looted repeatedly. Just exit the system and come back, or dock at a local base and go back out, and they're ready to be looted again. I suspect the 0 value on these wrecks is an oversight, rather than intentional; however, the normal behavior of wrecks is to flag your user profile indicating that you have "visited" a wreck already. This is shown by making "visit = 0" on the wrecks in a system, and those wrecks won't drop their loot if you've already looted them; but if you haven't already looted them, they start dropping their loot repeatedly, because the game no longer flags your profile due to the setting on the wreck.

This part is just a theory. The flag for a particular wreck in the player profile may have a "timeout", meaning it will clear after awhile, allowing the wreck to be looted again. It may also be the case that attempting to loot a wreck will restart the clock, even when you get nothing. This would explain why wrecks do respawn their loot, but it can seem like they don't: you have to wait. I don't know how long this timeout might be, but I've read other posts saying it's a week (I assume that's real-time, and not game-time).

As for MP games, I think the server has an internal flag for the wrecks, which it will "pass on" to anyone attempting to loot them; so the looting of one player can affect the looting of another. But restarting the server only resets its internal flag; and once a player has the "visited" flag in their profile, the game will go by that. Thus, restarting the server only helps a player that hasn't already tried to loot a wreck.
