spherical

Wed Apr 23, 2003 9:09 am by spherical

I hooked my laptop together with my desktop computer into a LAN yesterday. I found few interesting things:
- Started server and game on the desktop (nothing run on my laptop) and start playing. During the game play (in the middle of the mission), I disconnected my laptop (pulled out network cable) and guala, server crashed even that nothing was running on the desktop (This is because windows "disables" network if no connection is established). Unable to use account, which was at that time used for playing (server crashes at login). If I created a new account, it worked perfectly.
- Another case. Played game, intentionally killed the game in the middle of the mission. Server crashed, unable to use the account used at the time of playing (server crashes at login). New account works perfectly
- Another case. If I'm killed in the middle of the mission, I'm unable to use that account (server crashes when trying to log in). New account works fine.

Bottom line: Something is wrong with the account file if user doesn't land on whatever base after the mission (get killed, lost connection). When you land, the file is written correctly. (state machine problem?) As I expected, there is something wrong with the parsing of account file when user is trying to log in. (at least in my case)

Possible solution: When you successfully finish the mission, make a backup copy of your account file (server side). Next time when server crashes (or game) and your server crash at login, restore the account file. I was unable to use any account after a crash - needed to create a new one.

This isn't a nice solution 'cause you have to switch (alt+tab) to desktop and make a copy each time you finish the mission, thus you have to be a server administrator.

It would be nice to have a tool for that.

Hope this helps...
