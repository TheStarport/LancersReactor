Stinger,
I am going to assume something, and that is;
"the flmon program to restart your server" really means the Freelancer Server Program, not rebooting the machine it is running on. I say this because I got confused by your using the term reboot: "or do a manual reboot to ensure hidden wreck"

Here's a rundown of how and when on Browclops: 90% of the time there is no physical reboot of the machine itself. I have not experienced any memory leaks or other problems that would warrant a reboot of the machine. When it's time to restart Freelancer Server program I stop the FLAdmin program first, to prevent multiple instances of the Freelancer Server program from starting up, as has happened if I was slow. Then I stop the Freelancer Server program. I do the account backups, then start the Freelancer Server program. I adjust the Description info to reflect the date and time, then hit OK (or whatever it is). Freelancer Server program runs at about 33msec for a few seconds, then drops to 0msec. I adjust the information in the Server News to reflect the restart time and other basic info. Then I start the FLAdmin program. I do this about twice a day, sometimes three if I see lots of newer people.

Roughly three-five times a week I will see that the FLAdmin program has restarted the Freelancer Server program at some point. I do nothing but change the information in the Server News to reflect the latest restart time.

As to campers slowing down the program restart process, I've never seen anything more than about 10 seconds for it to decide 1)I'm ok to shut down or 2)I suck and want to notify Microsoft. It has never hung in the traditional sense (the endless wait)

Hope this helps a little at least.

Browclops
P.S. I changed all references to FLMON to the correct program, FLAdmin v1.0.4, after I woke up and realized I had specified the wrong program. - Browclops
