WatercoolerWarrior

Wed Dec 29, 2004 2:01 pm by WatercoolerWarrior

A few quick bits of feedback on this. Overall, this is a highly useful tool.

+ Persistent initial values
It would be useful for the trade route calculator to read settings from a location such as the registry or a co-located TradeRouteCalculator.ini file. Specifically, if we could preserve money/cargo/source/destination, or at least the first 2, it would make fast use a little easier.

+ Startup time?
I would like faster startup, BUT I can see reasons not to. Obviously you would need to cache data somehow; your stored dataset would no doubt be custom; and then some people would forget to refresh when they need to.

+ Proliferating debug files
I have about 20 of these now. Would it be possible to have it use a standard TradeRouteCalculator.mjf each time?

+ feedback prompts and feedback itself
The feedback prompts get a little annoying after a while, but I'm not sure how this could be handled better. I've written specialty tools that are downloaded by hundreds of people (not FL stuff) and only rarely get feedback on how good it is or errors it has.
Perhaps a saved setting that turns off feedback prompting after some comment has been submitted?

+ Asgard mod: Maladan trades not showing
While playing with this against the Asgard 2.2 mod, I noticed that Maladan station's trades do not show up. However, they don't show up in-game for routes, either!
So this may be a "feature" not a bug - behavior identical to in-game trade route information that is available.
