Toolkit 1.3 Progress Update

Build 4 will be released to beta-testers sometime over the next 24 hours. Right now, there are still quite a few issues to be resolved before the final Release Candidate is given to the beta-test team, but I've been making steady progress.

Most of the weapon changes have been finalized, with a few niggling issues pertaining to Missiles.

Ships are going to get re-balanced using new formulae before I release Build 5. Right now, most of the Infocards are completely wrong- this will get fixed as I move towards what will (hopefully) be the final build of the mod.

All Factions have had their Loadouts, Npcships, and Faction_Prop entries re-written from scratch.

New, 100% custom SURs will be getting tested in Build 5.

Special effects, such as my new Thruster effects, will be finalized soon.

Various custom ships, weapon models, and other things are gradually being integrated into the builds. Expect to see a lot of Jsncalif's outstanding models in the final mod, along with some new Argh ships to address some "holes" in the game design.

Encounter code has been rebuilt from scratch, allowing for much more varied and interesting Encounters througout Sirius. I don't think any other mod has added this many changes to the way that Encounters work.

Argh

Mon Nov 28, 2005 7:12 pm by Argh

I can always use more beta-testers. Just shoot me an email. Keep in mind, though, that you might want to try Toolkit 1.2 before you drool- 1.3 will be a much-improved version, but 1.2 gives you a pretty good feeling for how the Toolkit series has been designed.

Until I got going with 1.3, I pretty much left the core game design alone, other than fixing the things that are, to me, obviously wrong with the original game design- things like weapon velocities that don't match, the arbitrary levels assigned to equipment and weapons that made a lot of ships completely unplayable in serious MP combat, etc., etc., etc.

1.3 features some pretty important game-design changes that will take some getting used to, but will hopefully get used by modders- the whole objective here, after all, has been to make a better starting codebase, so that modders can start FL mods from a good, well-organized codebase, instead of the messes that DA left behind. FL is a remarkably bug-free game, despite how it was built, but the objective of this project is to make it so that even more people can jump right in and start building things for FL, without spending as long wandering through the code
