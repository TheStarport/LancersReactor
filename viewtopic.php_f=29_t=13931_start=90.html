Neph,

Hey, that's what happens when you create the files at 7AM after being up for two full days with no sleep. *LOL* Told you my mind wasn't working right. At any rate, figured out how the X-Y crap works on the map positions too. If anyone else needs help in that area here's the jist of it.

*****************TUTORIAL*******************

MAP LOCATIONS (pos = X, Y, Z)

0, 0, 0 = Center of the map
+X, 0, +Z = Bottom Right of the map
+X, 0, -Z = Bottom Left of the map
-X, 0, +Z = Top Right of the map
-X, 0, -Z = Top Left of the map

You can mess around with the Y position if you wish but I find that only creates problems in locating what you've added to the map.

As well, in the Liberty system one box seems to be 33000 wide and 28000 tall, incase your having troubles with the measurements. For other systems you have to check "Universe.ini" in the Universe directory to find out the "NavMapScale", which is the scale of the map compaired to Liberty (default). In other words if you have a "NavMapScale=2" then you would multiply the grid dimentions (33000 wide, 28000 tall) by 2 for that system. (atleast I think multiply)
