Ok I normally would not post to a thread like this, but this is why so many new modders are asking so many questions they should not need to.

We'll start with the title "** TUTORIAL ** The even simpler way ids names!" nothing in this thread deals with ids names, just an unnecessary process of placing an empty .dll where FLexplorer creates a blank .dll for you.

The creator of this thread should have searched for why his ids names were not showing up and he would have found out there is an easy way to properly link the .dll to the mod by placing the new .dll in the exe folder, than in the freelancer config file found in the exe folder under the resources section adding the line DLL = my_mod.dll ; discription of the dll

In the end this tutorial and ones like it should be deleted as it only clutters the forum adding usless information and will only result in new modders asking simple questions they should be able to find answers for themselves.

Don't get me wrong Unib5 is trying and I can not blame a guy for that but when he fails then it needs to be dealt with.
