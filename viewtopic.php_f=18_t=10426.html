Making Jump-Gates more... Jump-Gate-ish.

I got a little irritated with the fact that Jumpgates are supposed to allow "instantaneous travel" between systems, and yet, takes 5-10 seconds to traverse. not quite instantaneous. Understandably, it saves mid-jump gate (to the auto-save.) i had originally over-shot the mark *resulting in a very interesting error to the game. it locked the system completely until it naturally played through the jump-gate, and then crashed*

This is apparently the shortest time i can get it to without crashing:

*note: In order for it to work, you need to edit them all.*

in Data\FX you'll see "jumpeffect.ini"
these are the effects for all the different jump-gates. jump-holes, jump-gates, etc.

[JumpGateEffect
nickname=jump_effect_hole
glow_ring_effect=wormholeactivation
glow_ring_hp=HpFX1
glow_create_time=1
jump_out_time=1.5 ; this is the time it takes to "jump out" of the current system, into the tunnel.
jump_out_tunnel_time=2 ; how long you are in the "out" tunnel before it saves and switches to the in-tunnel.
jump_in_tunnel_time=2 ; the obverse of above.
jump_in_time=1 ; see above
kill_time_before_done=0.6 ; this is apparently the kill-switch. i managed to set it to .35 and it worked, but was not reliable, it would occasionally crash. If anyone else is patient enough to deal with the crashes, please find the limit? Apparently this is the "Turn off the Tunnel" switch if you have not met the above.

With all the settings as such, it takes about 2-3 seconds on my PC to pop through a jump-gate. it makes them MUCH more manageable.

jump_tunnel_effect=jump_tunnel_interior_player
jump_tunnel=gate_tunnel_bretonia
jump_ambient=25, 25, 50
jump_background_color=255, 255, 255

That's what i figured as well. Hopefully I can cut it down a bit more.

For those of you who want it to appear you're going faster:
This is another "edit each setting" scenario:

in \data\fx\gate_tunnel.ini
[gate_tunnel
nickname=default_jump_gate
write_depth_buffer=0 ;I wonder about this, possibly for high-end videocards?
num_spline_control_points=9 ; ahh. splines. you sorta see this when you're "flying" i tried increasing it, it didn't really change the effect.
x_range=500; Messing with these not recommended. can create
y_range=500; neat effects though. these are the width, height, and
z_range=20000; depth of the Jump-gate.
min_radius=50
max_radius=100
far_radius_factor=0
min_speed=2;important fun. setting these too high causes problems though.
max_speed=2;
time_to_max_speed=1
fade_distance=0.1
near_alpha=1
far_alpha=1
num_t_steps=200
num_s_steps=12
min_rotation=3;makes the tunnel spin.
max_rotation=-3;at 3, -3 it doesn't really move much. set it higher.
min_rgb=0, 0, 30
max_rgb=0, 0, 100

realize the defaults to min_speed and max speed are:
min_speed=.003
max_speed=.08
time_to_max_speed=5

setting it to 2, 2, and 1 is about the highest you can get, you don't get to "enjoy" the jump-gate, but you sure go through.

might be able to do 3. but 3/5/1 is not possible. crashes.
