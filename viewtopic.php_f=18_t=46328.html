Hexplorer is fine, and so are several other hex editors - as long as you remember when you change hex values you have to reverse the bytes; for example if you want to enter 09AB CDEF then you have to enter it as EFCD AB09 because the bytes are higher value to the right, not to the left.

I notice your system music is "music_ba_space" - is this correct, or do you want "music_br_space"? I saw your post on music so it's probably correct, I guess you have made new music for your new system?

One other thing I notice is:-
<pre><font size=1 face=Courier>
100000000f in Freelancer.exe, 0x1D15B8 = square of max distance that a ship can be selected
0x1D15B8 = 1.1 (*3)
6250000f in server.dll, 0x84ADC = square of NPC disappear distance in MP
0x86AEC = 1.1 (*3)
25000000f in server.dll, 0x84AE0 = square of player disappear distance in MP
0x86AF0 = 1.1 (*3)
100000000f in server.dll, 0x84AE4 = square of grouped player disappear distance in MP
0x86AF4 = 1.1 (*3)</font></pre>
You should perhaps make sure the values are proper square roots, not just *3.
So if you want 20000 then your entry should be 20000 * 20000 = 400000000. If it was 400000000 and you * 3 then new value = 1200000000, the square root of that is 34641.01615 and not an integer. So its better to calculate 60000 * 60000 = 3600000000 to get 3 times greater distance.
And these numbers with f at the end are not plain hex conversions, you must use single-precision Floating Point values:-

30000 ^2 = 900000000 = 4E5693A4 ; enter as A4 93 56 4E
7500 ^2 = 56250000 = 4C5693A4 ; enter as A4 93 56 4C
15000 ^2 = 225000000 = 4D5693A4 ; enter as A4 93 56 4D

Hope this is helpful.
