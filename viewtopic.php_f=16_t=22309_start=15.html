Below is a list of hash numbers that represent jump holes. They are members of <pre><font size=1 face=Courier>visit = <hash number >, 1 </font></pre> and <pre><font size=1 face=Courier>holes_visited = <hash number > </font></pre> lines in the *.fl files.

If a hash number is listed in the first group of lines, then the corresponding jump hole appears in the Nav-Map.
If it's listed in the second (i.e. holes_visited ), then the Neural Net will regard it as "found".

Example:

By assigning specific value for system and base , and deleting the location line, you can be at any base / station / planet of your choice in the Freelancer universe.
The value for system consists of two letters representing the house / world followed by the identifier number of the system within that house / world.
The value for base consists of the value of the system above followed by an underscore, the identifier number of the base / station / planet within the system, an underscore, and the word "Base".

For example, if you want to be on Atka Research Station, then the *.fl file should contain the following lines
<pre><font size=1 face=Courier>system = Bw06
base = Bw06_02_Base </font></pre> and no
<pre><font size=1 face=Courier>location = <10-digit hash number > </font></pre> line.
Note: You may keep the location parameter if you know exactly the correct value for it, since I haven't figured out the hash numbers for location yet.

The table below can help in determining other bases / stations / planets.
• The houses / worlds are written in CAPITAL letters followed by its two-letter identifier enclosed in parenthesis.
• A system name is also followed by its identifier number enclosed in parenthesis.
• A base / station / planet is numbered according to its identifier.
