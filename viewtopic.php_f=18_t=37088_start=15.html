Sir Clarence

Wed Mar 16, 2005 3:58 am by Sir Clarence

When saving Infocards with Fledids after using explorer make sure your files are not writeprotectet. I needed about 2 hours until i found out that the infocards.dll is writeprotectet and thats why fledids can't write the string. There's no errormessage showing up.
Make sure you are using a free ids string

1. Disable writeprotection for the whole freelancer directorie
2. Find out the first free ids string
- Thats easy just open fledids
- in the field ResID to ids_name/ids_info choose InfoCards.dll
- now find the first free Resource ID
Example: in my InfoCards.dll it is Resource ID: 710
- push convert
- copy the converted id
Example Res.ID 710 is converted to 66246
- paste it into ids_info and push Get ids_info if the field is blank you are to far
- repeat the first steps going down with the Res.ID (710 after that probably 600)
until you find the last filled entrie
- enter you InfoCard an Res.ID after that number
Example last filled is Res.ID 709 take 710
- why??? I discoverd that leaving blank fields, often causes Freelancer to crash or
it won't find the right string
- write down the Res.ID every new Infocard had to follow. don't leave free
strings.

Thats how i make new Infocards. I hope it helped you.
