Audio/ Fraunhoffer MPEG-Layer 3 Codec Fix

Okay, enough's enough. Those who are still having problems related to the MP3 codec, email me at l4g4 AT tbi.co.id (replace all occurances of 4 with a so that it reads laga) and I will email you a custom installer that should fix this.

Obviously I can't do this forever, so PLEASE - someone at lancersreactor, can this get hosted on the network?

It's a windows INF based executable installer, filename "Radium MP3 Codec v1.263.exe", 434 KB (444,672 bytes). It installs both l3codeca.acm (Advanced, v1.9) and l3codecp.acm (Professional up to 320 kbps, v1.263).


For those who aren't scared of editing system files and use windows 95/8/se/me, look at the file, C:\Windows\System.ini, and scroll down to the segment listed as [Drivers32. It should look like this, though probably alot shorter and not anywhere near as organized :

[drivers32
MSACM.imaadpcm=imaadp32.acm
MSACM.msadpcm=msadp32.acm
MSACM.msaudio1=msaud32.acm
MSACM.lhacm=lhacm.acm
MSACM.msgsm610=msgsm32.acm
MSACM.msg711=msg711.acm
MSACM.trspch=tssoft32.acm
MSACM.IMC=IMC32.acm
MSACM.iac2=IAC25_32.AX
MSACM.divxa32=DivXa32.acm
MSACM.sl_anet=sl_anet.acm
MSACM.msg723=msg723.acm
MSACM.L3ACM=L3CODECA.ACM
MSACM.L3CODEC=L3CODECP.ACM
VIDC.DIVX=DivX.dll
VIDC.IV31=IR32_32.DLL
VIDC.IV32=IR32_32.DLL
VIDC.IV41=IR41_32.AX
VIDC.IV50=ir50_32.dll
VIDC.MSVC=msvidc32.dll
VIDC.MRLE=msrle32.dll
VIDC.MPG4=MPG4c32.dll
VIDC.MP42=MPG4c32.dll
VIDC.MP43=MPG4c32.dll
VIDC.YUY2=msyuv.dll
VIDC.UYVY=msyuv.dll
VIDC.YVYU=msyuv.dll
VIDC.I263=I263_32.drv
VIDC.I420=I263_32.drv
VIDC.DIV3=Div3c32.dll
VIDC.DIV4=Div4c32.dll
VIDC.M263=msh263.drv
VIDC.M261=msh261.drv
VIDC.HFYU=HuffYUV.dll
VIDC.xvid=xvid.dll

Ignore EVERYTHING except for :

MSACM.L3ACM=L3CODECA.ACM
MSACM.L3CODEC=L3CODECP.ACM

If you have both, and if you want both to function (ie, if you use MP3 rippers or edit videos with Virtualdub), make sure that msacm.l3acm is FIRST, in other words it comes BEFORE msacm.l3codec.

Otherwise, DELETE ALL REFERENCES TO L3CODECP.ACM.

If you have only one, make sure it points to L3CODECA.ACM.

IT DOES NOT MATTER WHETHER IT SAYS MSACM.L3ACM OR MSACM.L3CODEC.
