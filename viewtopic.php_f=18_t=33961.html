bane42

Mon Sep 06, 2004 3:53 pm by bane42

i wanted to do this also for a mini total conv mod
and heres what I did to start out with a mp only, one system universe.

I have left the start system as Li01 and the start base as li01_01 just rewriting the code
and I am using all the same factions as before

data folder
initialworld.ini commented out ( i.e.) ; in front of all the locked gates entries

Equipment folder
market_commodities.ini zap all old entries and replaced with my new li01 code only
market_misc.ini zap all old entries and replaced with my new li01 code only
market_ships.ini zap all old entries and replaced with my new li01 code only

Interface folder
knowledgemap.ini zapped all entries relating to systems (will replace later) so the only thing left is faction data

missons folder
mbases.ini zapped all the old entries and replaced with my new code for li01 bases only

universe folder
missoncreatedsolars.ini zapped all entries
shortest_illegal_path.ini zapped all entries except li01, li01, li01
shortest_legal_path.ini zapped all entries except li01, li01, li01
systems_shortest_path.ini zapped all entries except li01, li01, li01
universe.ini zapped all entries except the intro1_base, intro2_base, intro3_base entries and added in my new li01 base and system entries

Li01 folder in systems
replaced the files as appropriate with my new details

exe folder
mpnewcharacter.fl at the bottom i commented out all the system visit entries
(e.g.) visit = 2745692751,1 etc.

bundle this up into a mod and away you go

I am not sure of all of this is necessary, but it took some time to get it to work.
you will get some base not found messages on the flserver.exe console view when u boot up the server but thats ok. The messages reference the news.ini in the missons folder (if you zap that file the server will crash and fl wont start up)

Bane
