Bejaymac

Sun Mar 25, 2007 1:11 pm by Bejaymac

Try CTD's by the barrel load for both clients and servers, the mbases is the only file AFAIK that has to be perfect , the slightest mistake, typo, extra base or duplicate base will destroy all encounters in your systems. No NPC's will dock and every patrol path will cause nothing but CTD's when your within spawning range (about 5k).

@ Louva-Deus, having debugged a few mods over the past couple of years I can say that FLE does leave crap behind. Take Ruppetthemuppet's problem as a classic example, build a base and fully kit it out and then some time later delete it, most of the time FLE will delete every thing for the base, but ocasionally FLE will forget to delete the mBases entry. Then if you build a new base in that system FLE will use the deleted bases number, but it won't use the existing mBases entry, instead it makes another one creating a duplicate entry and causing nothing but problems.
