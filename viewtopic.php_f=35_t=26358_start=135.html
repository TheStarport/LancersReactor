LightRogue

Sat Apr 03, 2004 7:10 pm by LightRogue

Looks like this project may be on death's doorstep. Too bad. I know Freelancer isn't the hottest new game in town, but it is still a cool game, and a lot of fun to play with when you can edit it. I think it still has a lot of life left in it, but only if those supporting it continue to do so. When the tool pushers go away, so does the editing community. If the current developer is asleep at the wheel, it is time to find a new developer. I've seen his name mentioned, but he's had the tool for months and has not released a new version of his own yet. Since he seems to be on vacation, maybe it is time to put the tool into capable hands - somebody who WANTS to deal with the project.

As to the last two posts, you guys have to do one simple thing to solve those problems. You have to ignore what the program installer wants to do and remove the FREXP files from its own folder. The program will NOT build a mod when run from inside of its own folder. You have to cut the files and paste them into the Freelancer EXE folder - yes, a bunch of loose files in your Freelancer EXE folder!

Two things to remember: 1) you will have no way of telling your Explorer files from your Freelancer game files; and 2) you cannot UNINSTALL FREXP once you do this because the installer/uninstaller will delete every file in the folder it is in, including the game files.

However, dropping the FREXP files into your EXE folder will eliminate the "Cannot find C:\Temp" error. Your files will save and if you have Mod Manager installed properly, it will work with Mod Manager and you will have a working MOD that can be used in the Mod Manager.

FREXP will RUN properly from inside its own folder. That was my mistake the first time I installed it. I assumed that because it runs and I can edit a system, everything is fine. But I was unable to save my work the way it is intended to save. I always got that error that says there is a problem saving the files. Then when I reopened my mod, names of things were missing and most of my technical data was also missing. When I moved the FREXP files out of their own folder into the EXE folder, my mod saved as a mod, and the custom dll file was created for me. The names of objects are automatically stored in the dll file and when you run the mod, the names show up in the game.

I don't come in here very often. Posts are few and far between and there have been no updates of the tool since what's-his-name took over the project. But here is a suggestion: include a ZOOM feature for close work. Setting up Tradelanes works, but getting the ends where you want them is something that requires closer detail levels.
