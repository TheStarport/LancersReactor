Editing Multiplayer account files directly?

Recently had a bad server crash, and two of my Kills leaders had their accounts corrupted. I was able to restore their accounts from a backup, and using the wonderful Iconcross server operator, got their money and equip back to current. But there is no way to edit kills with that program.

The multiplayer account file is encoded with something that I can not figure out how to get into. Tried BiniQDU but to no avail.

Has anyone figured out how to decode and edit a multiplayer account directly in the file, or has anyone run across a program or tweak that allows you to edit the kill count for multiplayer?

There was one utility that could decode the fl files to a text file and re-encode a few months ago.

But Prevention rather than cure, you should be taking backups weekly. I hold the backup zips dating back 1 month, so if theres a problem like that its easily fixed.

Also note the crash probably happened for one reason, that pair of players where F1'ing quickly while mining NPC's outside a base so they could sell the items in the base, it's a surefire server crasher. I think the reason is the Server exe holds the player file in its memory for about 30 seconds after a player quits, so eventually they will time the F1 switch to when the Server Physically Writes to the FL file and so corrupting it. So watch for players doing it too often and stop them.

You will have to find the limit of it to recognise it, so only way it to do it your self

In alphabetical order, probably best to start at bottom:
o FL Saved Game Decoder/Encoder (cmd)
o FLS1 (cmd)
o Freelancer Save Editor (GUI)
o Freelancer Simple Save Editor 1.0 (GUI)

There are probably others about, but these will enable to decode/encode the files at least.
