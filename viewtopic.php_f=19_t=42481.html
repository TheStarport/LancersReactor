Concerns about my Freelancer server

Hi all.
I am currently running a small server called "Narnia."
I have set it up to start in the Omega-5 system as a Corsair with a Legionnaire ship on Cadiz base.

I've been having a few troubles though. I've got the server on the net correctly, eg. port forwarding and all that, but I've been concerned about issues happening with the playerfiles and IFSO mk. V.

Let me explain. For example, a lot of playerfiles have people reported as being docked at Cadiz base in the Omega-41 system. There is no Cadiz in Omega-41 - it's in Omega-5! When I look at the .fl file in the save editor, I can see that something's gone wrong:

System = Ew03
Base = Bw02_01_Base

Which is totally wrong, as the system should show Bw02. This is not a problem with my mpnewcharacter.fl file as it is correct in there and this confusion only happens with a player that's been flying around - the server's doing it. Maybe it's a cheat? Is someone trying to exploit my server? Hmmm...probably not, but anyway.

Also, IFSO crashes alot. Alot. But only while there's people on the server. It's a standard VB runtime error "Bad file name or number" or something. I have also got a "Permission denied." Does anyone know of any issues with IFSO? I've checked regional settings, that doesn't seem to be an issue.
It's really bad that IFSO's crashing because if the server crashes, then the server operator won't be there to bring the server back up.

Also, one more thing. In my mpnewcharacter.fl I'm modifying the equipment so people start with a Legionnaire, but I'm not using hashcodes for the equipment which seem to be used on all the server-generated .fl files. FLServer doesn't seem to mind but the IFSO error log is flooded with issues like "Unknown EQUIPPED item found for player [haha - unknown hashcode was [ge_pf_engi" and so on. Any comments?

I would appreciate it if anyone that knows anything about these quirks that are going on with my server could give me a few hints.

Thanks in advance,
Jeremy
