Thu Aug 05, 2004 3:07 pm by Earendil

There is lots of fixes posted over and over for a non-connection with the FL global list server. Please consider the search function in the future.

Basically it is usually a DirectPlay problem, which is part of DirectX. Try installing/reinstalling with the latest version v9.0b. Use DxDiag.exe to diagnose DirectX. It could also be an OUTGOING firewall, like ZoneAlarm, as LaserMeat mentions.

Issues with WinXP's firewall now and in the future when M$'s massive patch turns the firewall on by default, are only prevelant when running an FL server. But because WinXP's firewall has UPnP it self-configure's itself for the one INCOMING port flserver.exe (Fl server executable) needs.

The patch should ONLY be used for a server install, NOT a client install. Even if you do there is no indication, FL version remains at 1.0. I believe the SDK, the so-called FL v1.4 incorporates parts of the patch (I could be wrong, please search for FL SDK v1.4 in TLR downloads for details) but it too does not change FL displayed version.
