Try reducing the hold size to something lower...it could simply be that the number you picked is too high to work, hence you're crashing. I've edited a couple of my ships to have larger hold sizes...3000 - 5000 is way more than enough, and it works great. I've got a Starflier right now that has 3 class 10 weapon slots, upto class 8 shields, and 3000 unit cargo hold...works great, cept there's bugs in multiplayer...nanobots will NOT activate...any hull damage taken cannot be repaired, so I upped the hull stats too...not that it seems to make a difference, as once I'm outta shield bats and have no hull left, I go boom...

Anyway, try using a more 'reasonable' cargo hold size, see if that helps. It could also be that you inadvertently put an extraneous character in that line, which the game cannot parse...recheck your ini file to see if that is the case. Hope everything works for you.

Thu Apr 03, 2003 5:54 pm by admiral_tolwyn

delete it and re-load your backups, then do this exactly as i say.

shiparch.ini should be unreadable right now. if not go here and put the origional in. again, you should not be able to read it with notepad - at least, not so it makes any sense.

now open up bini and open shiparch. now click "save as" and select "INI" this is different from "biniqdu". once you have done this, microsoft will be able to read your file even though it has been decoded. now open up your file, and change the cargo to an integer (no decimals ) between 1 and 9999. once you add the 5th digit, things start bugging out.

hope i helped, - tolwyn -
