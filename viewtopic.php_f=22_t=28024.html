Earendil

Sun Feb 01, 2004 6:00 pm by Earendil

Haha! I have been running AkaTaka's FLAcctBrowser and FLConnMon since I got both in his beta test! Excellent programs both. The beauty part of FLAcctBrowser is it records ALL characters ever run on the server. That's regardless if they were created one second and deleted the next. Very handy when tracking players using character names as abuse and thinking they get away with it through a delete! Plus FLConnMon links to the XML file FLAcctBrowser makes of all the characters so it lists the character name to IP address.

It is not slow at all for me. That is because FLAcctBrowser uses a decrypter on flserver.com so no CPU cycles are used. AkaTaka says flserver.com should be up for at least the year. Neither do I see any percent of my server CPU utilization being used for FLConnMon. As I've said before, excellent work AkaTaka!

EDIT: Didn't see Pinger's last post. There is some bandwidth usage for the flserver.com decrypting but it is minimal. Although I am at my bandwidth limit there never seems to be any indication in the game from the bandwidth it uses. I set mine to update automatically every 12 hrs so it's not a huge drain anyway.
