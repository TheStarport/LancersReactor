Systems_shortest.ini

specifies the shortest route on a system level between systems from a given system...

ie Li01, Rh01, Li01, xxxx, xxxx, xxxx, xxxx, Rh01

the first is origin, second is destination, subsequent is shortest path between those sytems through systems xxxx

ok so its a bit clunky but thats how it is
as far as im aware..

as for intra system pathing, i havent quite sussed that one, but i think it relies on internal routines as it works in systems that i have created.. so long as everything is labeled right it seems to work fine
