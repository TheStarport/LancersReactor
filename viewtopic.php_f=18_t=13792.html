*POLL* Future Features For FLMM

Now that FLMM has advanced enough, it's time to consider some advanced features:
#1: Making FLMM create a resource dll based on data in a mod script?
#2: Making a tool to automatically create mod scripts by comparing your edited ini files to the original ini files?
#3: Letting mods that modify the same ini file be compatible, if possible?

Which feature would you most like to see implemented first? Are features #1 and/or #3 un-needed? I'm leaning toward #2 right now, because I'm not sure I need to implement #1 (FLMM already automatically installs custom resource dlls; you just have to make the dll first), and I don't think many people would really want #3.

Give me your feedback! And, if you can think of anything else that FLMM should have/do, let me know!
