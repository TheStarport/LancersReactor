Wed Jul 13, 2005 6:56 am by Vichome

I had the same problem. There is only one solution:
1) If you already configured IFSO to autostart FL-server, delete IFSO config file FLServerOperator.ini
2) Start IFSO and press "Cancel" at startup (don't let IFSO to control the server)

At the page "Configuration/IFSO Settings":

3) Set the path to your flserver.exe ang flserver.cfg (if the program ask you to do this)
4) The text in the "FLServer Window Name" should be the EXACT copy of the window name of your FLServer. Edit it if needed.
For example: "FLServer - Version 2.28 Build 11 - FLServer" (my EVO 2.28 server)
5) Change the time and date format, if it not match your Windows time and date format.
6) Press "Save Program Configuration"
7) Quit and restart IFSO - now you can start FL-server and not freeze IFSO
