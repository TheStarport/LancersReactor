Hey Burzelmann, i've got a few things for you to check.
First off, do you get base population at the bases you have added new factions for? If you don't, its likely there is an unwanted entry in mbase that refers to a base that no longer exists. Mbase is the kind of file where if you have one error, all of the entries after that are screwed regardless of wether they are ok or not.
To test, you can copy your testing base to the top of the file and to fix, use a version of opensp to spawn at progressively higher bases.

Next one, vigretteparams, did you add all of the custom factions to this file? You won't get any missions unless you do. You should add them to killable_solars too. The easiest way of doing it is to use the replace command and replace:
"fc_c_grp," with "fc_c_grp, AAA_grp, BBB_grp, CCC_grp," for unlawful and use gd_bh_grp for lawful.

Other than that, i would go a bit crazier with the vigrette zones inside the system. Don't put them too far away from the base (it dosen't like that) and try a combination of law, unlaw and unlaw, law. The "open" type as well can be good for getting things like destroy the base type missions.
Well, thats all i can think of for now. Post up to let me know how you are getting on.
