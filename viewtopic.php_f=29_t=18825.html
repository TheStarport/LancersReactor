***Tutorial*** 3D Studio Max to CMP

After you have made your model Export it as the .3DS filetype.
Now run Milkshape3D with HCl's FreeLancer CMP plugin for Milkshape3D installed and then go to File>Import>Autodesk 3DS. Now fund your file. Your model will now be loaded into Milkshape3D. HCl's CMP plugin lets you then export this model into a CMP.

Fri Jul 04, 2003 10:26 am by free spirit

3db files are stipped down cmp files(i.e. single v-mesh model files, not CoMPound models), used for various things like icon placement, sheild location,bases, characters. they seem to be how DA generated models and then make cmp files by combining these together for ships/larger objects (although in some cases DA break there own rules i.e. bars and make compund 3db files) .unlike the modeling comunity here which make single model cmp/v-mesh files (with no multiple lods). at this time it is a long laborious process to make either 3db or compund cmp(with lods) manually, but can be done (several people are working on this).

sur files are surfacemesh files. there are 2 types sheild -which is just a bubble around the ship and ship sur files - which are a simplified ship models. there is nothing yet that can encode these models/files. (Please note i didnt say decode here, as several people have stripped the data out and we have seen what the models/shapes look like, although the full information carried in these files has yet to be understood)
