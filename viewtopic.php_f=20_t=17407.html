Vanarion

Mon May 26, 2003 1:55 pm by Vanarion

I too would consider faulty RAM as the source of your problem. Random crashes often indicate that the RAM doesn't keep data correctly. This may sometimes only happen with certain programs that use a lot of memory. Other applications aren't affected for they don't address the faulty memory space. You should perhaps test the RAM using a program like CTRAMTEST from the german computer magazine C'T. Personaly i had quite some defect noname RAM-modules, so I guess this is not an uncommon problem. Often people regocnize it as a software problem and people often believe their system is somewhat unstable.
