Earendil

Fri Apr 01, 2005 6:55 pm by Earendil

There is one thing within IFSO's configuration window that mentions enabling auto-startup of flserver when IFSO is started. Yet this seems to be different than the issue with the configuration screen of flserver not closing. For both IFSO and FLAC this is automatic and built into the program. It clears these windows to ensure a smooth startup of flserver. There is no option to control it because it must be closed for flserver to finish starting up and get on the FL global list server.

If you are using IFSO and upon starting/restarting flserver this config window doesn't close then you have some other problem. You may not have the same identical character string that's in flserver's title ("FLServer - Version 1.0 Build..." in IFSO's config tab but from what I've seen it will not budge flserver let alone kill any flserver config window.
