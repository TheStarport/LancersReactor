IFSO known as Ioncross FL Operator is out and is an FL admin's dream. It takes care of many things for flserver.exe when running including restarting if flserver goes down. Just be extra special in setting it up like the correct flserver window title, date format, etc. Find it on your neighborhood friendly TLR website at this page.

It's certainly a potential problem with laptops that they may become overheated. They usually don't run flat out but instead speed step to reduce power consumption. With all the horsepower flserver needs it's a logical deduction that flserver will run the laptop flat out. On the other hand you say you don't have tons of players in no mass playing online at the same time so unless it's a weak laptop I doubt it has run at full speed for long. Hence I doubt the problem is with cooling unless there are other mitigating factors.

As for copying files over it depends what's set up on each 'puter, how it's connected and so forth. Quickest way is to ftp from one to another but an FTP server would have to be setup. Another way is just to remote connect in. Another way would be to email the files, sipped, to yourself and then webmail access them on the other 'puter. Another way is to hand carry the data yourself. Until more is known about the connections between the two 'puters then I can be more specific.

While technically all the FL MP saved game files (and SP files for that matter) are in the same format and thus interchangable. In actuality few of the other server's out there will allow you to email your MP saved game file (character) to allow them to install it for you. They would have to trust you to not have illegal things and such.

But if you can or it's between friends this is the right way to do it. Get the saved game file (I'll call these characters from now on) you want from some old server and send it to the new server admin. BEFORE you place it in the MP directories create a new character on the new server with the EXACT same character name. Then use IFSO or whatever you can to determine the file name where it's format is xx-xxxxxxxx.fl where the x's can be any hexidecimal number 0-9, a-f. Copy the new character's file name and use it to rename the file name of the old character from some old server. Then delete the new character. Then place the old character from another server relabelled to the new character's file name in the same directory the new character was. The end result is you have an old character from some old server with a file name created by the new server.

There is a good reason for all this and that's for admin and in game reasons. Sometimes the character name is not what flserver goes by but the file name. It can be most evident when you DON'T do the above procedure when you can create another character name on the new server that is the same name as the old character name. Then you have two of the same character names on one server where each have different file names. Not helpful especially on the admin side.

Hope that clears up your question.
