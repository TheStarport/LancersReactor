Slight server lag

I've recently decided to set up a new 24/7 server and, for the time being, it works smooth and well.

However, one of my members is complaining about sporadic yellow lag which screws NPCs and such. I'm trying to determine whether he is the only person with such problem (In which case it would most likely be a problem on his side), but I remember having similar problems playing over LAN (I'd get 1-2 secs of yellow lag every once in a while playing in LAN). I don't really see what could be wrong, since the box isn't a too bad one...

Pentium 4 3.4 Ghz HT
1 GB RAM in Dual-Channel

The server is strictly running FL and IFSO. I have Norton Antivirus 2007 running in the background for your usual virus checkup, but other than that nothing's taking up memory or CPU cycles. FLServer is also always monitoring a load of 0 to 2 msecs.

Any idea what could be the issue?

Good you mentioned Norton.

It's a heavy-weight and does slow down the response times, because it grabs processor cycles too often.

Try turning it off temporarily to prove if it is the cause or not, and if it is then try switching to Avast or some other package for a while. Avast is fine for me, I have not used others.

Ioncross FL Server Op can also cause lag if you have the "Autoupdate Players Online List" or "Ensure FL Server.exe is running every..." set too low. 30 seconds is good for both of these.

Hope this helps.

(I'd get 1-2 secs of yellow lag every once in a while playing in LAN).

thats a know bug when you play on your own LAN server.
The best indicator of serverlag is still simply the serverload.
As long as the serverload is not in the read-range (50+ msec) your FLServer runs fine.
If its still lagging, then there is an issue with the network connectivity, or some user related connection problems.
