I have some information on the "cannot find item on client request".
I assume that because nobody else has answered this question, they dont have the solution to the problem. Neither do I, but If I explain what I know, maybe someone else can find a solution/workaround...

I believe In Pilots population.ini, the "Repair Block" is the issue.

"Repair at damage percentage" is the point where the NPC will use his nans and bots, so if you set it to 0.90 it will use it at 90% remaining. The important thing here is that the NPC doesnt fully repair to 100%, it appears to repair only once each time it is at 90%. So, if you set it to 1% (0.01) the NPC uses the nans and bots when the shield/hull is down to 1%, however, even though they may have lots of nans and bots they only repair to 1%. The point here is that the NPC will never fully charge the shield or the hull, it will only charge it to the percentage that you set to start using them.

It also appears that the NPC actually don't know how many bots and nans they have. This can be demonstrated by giving a nomad lots of Bats/nans and then setting the repair at 1%. Now what happens is that the nomad uses these until its killed, but, because you do more damage (i was using nomad blasters on it) it doesnt use all the bats/nans up and then dies, result = no "cant find item on client request" errors. If you shoot until they expire of nans/bots, then you get the error message.

The problem comes when you get into a prelonged fight with an NPC or you include small amounts of nans and bots for low level NPC, with this you get a stream of these errors.

I think the reason for this is that while its "alive" its trying to use the nans and bots, but it doesnt know how many it has (hence the streaming of the errors). If you set the usage to 80% you get a huge amount of these errors than if you set it to 1%. In these exercises I adjusted the timing settings, and the lowest delay settings seem to produce the least amount of errors i.e. zero delay time settings
But, if you have to set the usage settings to 1%, you might as well not have nans and bots on the npc.

Where I became stuck was when I tried to backtrack with countermeasures.
I guessed that as it clearly knows how many of these its got surely there would be an answer to the problem and a fix, but I haven't been able to solve this yet.

I apologise in advance if people already know this, however, this error has been unanswered on the forum a couple of times.

Oh, this is very noteworthy...if you run a server and have these error messages spewing out, it DOES increase the work load on the server. If you get 30+ players on the server killing NPC and then revert to no nans and bots for the NPC there is significant change in server performance. Obviously the more players you have compound these issues.

Hope this helps..and I hope someone else can contribute to this also
