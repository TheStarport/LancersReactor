@Soulseeker:

I just reinstalled everything in my FL directory to clean, 1.0 release FL, installed FL SDK 1.5, deleted the Autosave in FLMM, and ran a New Game.

No CTD on my end. I'm running XP SP2 as well, so I dunno what to tell you, 'cept my best guess is that your installation's corrupt and you may want to download it again. If you'd be so kind... go get FLSpit (it's a utility for debugging FL mods) and run it right after re-creating the CTD event... and email me the results, they may help me pinpoint this and fix the issue (if there is one that's effecting more than one user).

@Bunny:

Um... there is a whole Forum section dedicated to such questions... but the short answer is that you need to look up the price of a ship Package entry, and change the price there. All of the new ships in the mod (including the capships and other stuff that have been put in with XML scripts) have [Good entries in the scripts that control their prices. Take a look at any of the example scripts to get an idea what I'm talking about, and go read the Tutorials and Newbie Questions threads

@Zessebel:

Read the documentation

Ok, I'll be nice... basically, you need to install FLMM (Freelancer Mod Manager, 1.31)... download and install the Freelancer SDK 1.3 (or 1.5)... and then install the mod. If the mod's opening up with Winzip or something else is kooky, then you're going to have to open it with FLMM manually, by right-clicking on the mod file and selecting "Open With" and then selecting Freelancer Mod Manager from the list.

Thu Jun 09, 2005 4:19 pm by Argh

Lastly, I've now played through the 2 takeoffs with King in the early game, and have not seen the errors you're reporting. Sorry, I think your installation of FL needs to be cleaned out. This happens when you've been playing a lot've mods, and occasionally have had trouble with them- they leave a lot of junk in your FL directories, which can cause various odd issues.

BTW... if you reinstall FL... delete the entire Freelancer directory (default installation path is c:\program files\microsoft games\freelancer\...), and then reinstall.

I keep a "fresh" install in a ZIP file and just unzip it whenever I want to "clean house", but I'm a coder, and kind've expect horrible things to happen on a regular basis
