Com_Trooper

Sun May 18, 2003 2:34 pm by Com_Trooper

You have discovered what I can only assume is a bug in the game. This can be fixed by editing your br04.ini file and looking for the entry titled "Br04_to_Br03_hole". At the "visit = " parameter below change the value from 128 to 0. Now if you go back to the jump hole in the game it should update your navmap.
I know of one other jump hole that has the same problem, so you might want to fix this one also while you're at it. Open br03.ini and go to the "Br03_to_Bw02_hole" header. Change the visit parameter to 0. This fixes the Cambridge jump hole that leads to Omega-5.
Please note that I didn't specifically explain how to uncompress ini files, there are plenty of threads about this in the editing forum. It's a mystery why these jump holes in particular have this updating problem because there are other jump holes with "visit = 128" parameters and they work just fine. Maybe someone with more editing experience could shed some light on what these values represent?
