Vertigo-1

Sun Aug 29, 2004 11:29 am by Vertigo-1

DS rocks. A tad slower than I'd like, but it's still an awesome program.

First up, some ideas:

- Searchable wrecks. At this point you have to open up every system, and mouse-over each wreck to see what it has (provided it even shows up on the system map). There should be some way to be able to search these for hardware or whatever.

- NOTES! Give us a way to mark a spot on the system map and tag a little note there. FL doesn't have it, and I'm still confused as to why my super-duper state-of-the-art, can navigate jumpholes computer, has left me without the ability to type some information in. So please, give us a way to do that!

- Show ammo in the equipment section. Ammo should be listed there (not just the missile launcher itself) because ammo is sold in many places where the launcher is not. What I want to be able to do is search for places to buy just the ammo.

- Show actual game data along with the infocard data. Take the Nomad fighter for example, the infocard has no information. No gun mounts, no armor, etc. If it's possible, the actual game data values should be shown, as well. Kinda like the extra stats part for ships, but more detailed.

Now for bugs or stuff I think should be changed:

- The ships parts. Ships isn't too bad, it's pretty fast and doesn't take long to update. But the last ship I've selected should be remembered. If I go back to the universe map, or click on a base from the Ships view, if I open the ships view again, it should take me back to the last ship I was viewing.

- Equipment area. Horribly slow with items that are sold on several bases. This should be fixed and streamlined or something, perhaps always stored in memory instead of having to search the bases over and over. Also, just like ships, the last equipment I was viewing should be selected if I close and reopen the equipment view.

- The Z axis. A few mods (Evolutions) for one, puts objects off the solar plane. FL really wasn't designed for "3D" space but it does work, takes a little getting used to. However, DS, like FL, totally ignores the Z axis. This should be added. For example, let's say you have a jumphole 5k below the solar plane in 7E. When you mouse-over the JH, it should say something like (X=E.5, Y=7.1, Z=-5k). This way you'd know that you have to go to 7E, and go straight down about 5k. Would be immensely helpful.

- Wrecks. I said above some wrecks don't show up in DS. Don't know why this happens. I search and search on my own, but I'm a detail freak. I like to have every system completely explored, all bases, JHs, wrecks, etc. So when I can't find something, it actually irritates me. That's when DS comes in handy. However, lots of wrecks don't show up on the map in DS (this is with a mod, btw, not vanilla freelancer).

That's about all I can think of right now. DataStorm is absolutely awesome, best program of its kind, so easy to use, and it just plain works. Also, I don't know how easy or hard my ideas would be to incorporate, but again, these are just thoughts. I'm not demanding them, though they'd definitely be awesome additions. Anyway keep up the great work!
