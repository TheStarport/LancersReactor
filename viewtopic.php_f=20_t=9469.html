You know, there's this file (I've since uninstalled it and am running Frontier: Elite II again) that goes through all these card types (just do a text search for Intel in the game install dir and you will find it), and then makes up all these disable rules. If someone could "open up" that file format for us, we could make our own entries, and thus solve some of our own problems. I have the elite but unmentioned SiS 730 processor on my mainboard, and get the blind buttons problem (and occasionally a transparency issue with the cursor), as well as some display artifacts. It suggests that the Z-buffer is involved in the text display, somehow. The thing is, if you don't do these layers right, they end up Z-fighting (yes, that's a technical term) and flipping text in and out, randomly. I suspect that the programmers did not use 'decal' overlays when they did this. It really is overkill to do them in polygons, but whatever... Heaven forbid Microsoft use DirectDraw to draw text, no, they have to use InDirect means.
