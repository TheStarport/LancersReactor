That doesn't make sense.. With a search, someone does a database query, and boom, there's the content they were looking for. Database query does = higher server load, but not that much.

Without a search, you have people A) browsing ALL of the posts (increases BW usage), B ) making duplicate threads (increases BW usage, increases database size considerably), and C) perhaps turning away regulars tired of reposting the same answers, and new guys unable to find their answers.

Now last I checked, Bandwidth costs more than server load. And if load is an issue, not sure if this board is capable, but phpBB and vB are capable of throttling server load to keep it under a specified limit. That might run into financial concerns again with license costs, but I thought I would point it out.

I've never run a board that had this much traffic, so there may be a technical issue I'm overlooking.
