Thanx Cibi, lukr and tylium for taking care of the answers, i was too busy working on the new models to even check this thread.
riverless, the new style of docking was necessary for the mod in its whole since it's uses capships. The secret of makin the docking scene visible is to set the docking distance for berths in solararch.ini bak to it's default values(40 or 60 I think). as in here:
change this
docking_sphere=berth, HpDockMountA, 800, Sc_open dock1
docking_sphere=berth, HpDockMountB, 800, Sc_open dock2
docking_sphere=berth, HpDockMountC, 800, Sc_open dock3
docking_sphere=berth, HpDockMountD, 800, Sc_open dock4

back to this:

docking_sphere=berth, HpDockMountA, 5, Sc_open dock1
docking_sphere=berth, HpDockMountB, 5, Sc_open dock2
docking_sphere=berth, HpDockMountC, 5, Sc_open dock3
docking_sphere=berth, HpDockMountD, 5, Sc_open dock4
default values here varied between 2.5 for small stations to 10 for large ones or planets, most vcommon values being 2, 2.5, 4.5, 5 and 10.

so edit the solararch and revert these values to defaults for all stations and planets dock rings to regain the docking sequence.

I'm also working of finding better surface files for the SW models, already the Ywing is much more vulnerbale to guns, and the Firespray as well.

Ro9ue

Thu Mar 04, 2004 6:16 pm by Ro9ue

If you download the patch, and you have the Freelancer Mod Manager v1.21, you should be able to double-click the patch from the folder into which you saved it.
It will then appear in the FLMM screen; it will be red (in conflict with FLRebalance v3.30), but tell FLMM to activate it by clicking patch and 'Activate mod' button. It's red b/c it writes over/"patches" the files in the Mod -- it's supposed to be red.
Then run FL, and it should be patched.
