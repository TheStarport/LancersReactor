Is that the actual Ping ie ICMP and not the server load? If that's the Server Load then you have cheats on board. Anything over about 300ms for a full server is usually someone cheating via a mod/trainer etc. But you have to figure out what is and isnt a load figure when someone is cheating.

Pinger, you should qualify your numbers with the server specs and the Internet connection. I, too, saw times like yours when I was running an Athlon 1800 FSB 133 MHz with my ADSL at 1.5M/384k (downlink/uplink bit rates). I had to limit max players to 19 and server load was consistantly over 100 msec near the max player limit. I upgraded the server to an Athlon 3200 FSB 200 MHz on the same Internet connection and the server load results were quite different for me:

4 players = under 5 msec
10-15 = 25 - 35 msec
16 to max 21 = up to 50 msec (35 msec is the yellow bar indication)

For me my limitation is the Internet connection, uplink specifically. I know because an increase in players doesn't significantly change the server load but I get tons of players telling me of yellow and even red lag while on the server. Thus 21 players is my max. In my case the upgraded server is a little too powerful for the Internet connection but that is fine because that leaves more CPU utilization for admin tools. Hehe! No more lag-a-minute indications because FLSO was updating it's online player list every minute!

BTW, overall modders would usually intrude on the server one at a time and they would essentially double all server load amounts.
