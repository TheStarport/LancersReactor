Actually I never "load" the SDK. All the SDK is (please correct me if I am wrong ) is all the ini files decrypted. Since there is no difference to flserver it makes no difference whether one or the other is in the game directories.

What I do is leave the original files and keep the SDK files nearby. Then when I need to modify them (for server-side mods of course) I start with an SDK file because it's unencrypted. I rename the original becuase you never know when you may have to retrace your steps.

If you do install both, it shouldn't matter as long as the SDK are files from a patched version of a FL server install.
